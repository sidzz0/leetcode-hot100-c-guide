## 建议刷题方式

1. 先看章节知识点回顾。
2. 每题先读生活化解释和暴力思路。
3. 自己尝试写代码。
4. 再看 C++ 题解。
5. 记录错因，隔 2-3 天复刷。

不要只看懂代码，要能讲清楚：

- 为什么想到这个方法
- 为什么这个方法正确
- 时间复杂度为什么是这样
- 哪些边界情况容易错
