## 一、哈希表知识点回顾

## 1. 哈希表到底解决什么问题

哈希表最核心的作用是：

> 把“查找一个东西是否存在”从 O(n) 尽量优化到平均 O(1)。

如果不用哈希表，很多题只能靠双层循环慢慢找。例如：

- 想知道数组里有没有某个数
- 想统计每个数字出现几次
- 想把具有相同特征的字符串分到同一组
- 想记录某个状态之前是否出现过

这些场景都很适合哈希表。

## 2. 生活化理解

哈希表可以理解成“带标签的储物柜”。

没有储物柜时，你要找一件东西，只能在一堆物品里翻。  
有储物柜后，每个物品都有一个编号，你直接根据编号打开柜子。

算法题里也是一样：

- 数字可以作为标签
- 字符串可以作为标签
- 排序后的字符串可以作为标签
- 计数数组转成的字符串也可以作为标签

只要你能为一个对象设计出稳定的 key，就能用哈希表快速查找或分组。

## 3. C++ 常用容器

### unordered_map

用于 key-value 映射。

你可以先把它理解成“字典”：

- 前面的类型是 key 的类型
- 后面的类型是 value 的类型
- 通过 key 可以快速找到对应的 value

例如：

```cpp
unordered_map<int, int> pos;
```

这句话的意思是：

- `unordered_map`：无序哈希表
- 第一个 `int`：key 是整数
- 第二个 `int`：value 也是整数
- `pos`：这个哈希表的名字

在“两数之和”里：

```cpp
unordered_map<int, int> pos;
```

表示：

> 用一个哈希表记录“某个数字”对应的“数组下标”。

比如：

```cpp
pos[2] = 0;
pos[7] = 1;
```

表示数字 `2` 在下标 `0` 出现过，数字 `7` 在下标 `1` 出现过。

### unordered_map 常用操作

```cpp
unordered_map<int, int> mp;
mp[3] = 10;
if (mp.count(3)) {
    cout << mp[3] << endl;
}
```

逐行解释：

```cpp
unordered_map<int, int> mp;
```

定义一个哈希表，key 是 `int`，value 也是 `int`。

```cpp
mp[3] = 10;
```

把 key 为 `3` 的位置赋值为 `10`。你可以理解成：

```text
3 -> 10
```

```cpp
mp.count(3)
```

判断 key `3` 是否存在。

- 如果存在，返回 `1`
- 如果不存在，返回 `0`

所以代码里经常这样写：

```cpp
if (mp.count(3)) {
    // 说明 key 3 存在
}
```

```cpp
mp[3]
```

取出 key `3` 对应的 value。

注意：

```cpp
mp[unknownKey]
```

如果 `unknownKey` 不存在，C++ 会自动插入这个 key，并给 value 一个默认值。  
所以如果你只是想判断某个 key 是否存在，优先用 `count()`，不要直接用 `mp[key]` 判断。

### 本章会出现的 unordered_map 写法

```cpp
unordered_map<int, int> pos;
```

用于记录：

```text
数字 -> 下标
```

例如：

```cpp
pos[nums[i]] = i;
```

意思是：

> 当前数字 `nums[i]` 在下标 `i` 出现过，把它记录下来。

```cpp
if (pos.count(need)) {
    return {pos[need], i};
}
```

意思是：

> 如果 `need` 这个数字之前出现过，就返回它之前的下标和当前下标。

常见用途：

- 记录数字到下标
- 统计出现次数
- 分组
- 记录前缀和出现次数

### unordered_set

用于快速判断某个元素是否存在。

你可以先把它理解成“只存 key、不存 value 的哈希表”。

```cpp
unordered_set<int> st;
```

这句话的意思是：

- `unordered_set`：无序哈希集合
- `int`：集合里存的是整数
- `st`：这个集合的名字

它只关心一个数是否存在，不关心它对应什么值。

### unordered_set 常用操作

```cpp
unordered_set<int> st;
st.insert(5);
if (st.count(5)) {
    cout << "exists" << endl;
}
```

逐行解释：

```cpp
st.insert(5);
```

把数字 `5` 放入集合。

```cpp
st.count(5)
```

判断 `5` 是否存在。

- 存在返回 `1`
- 不存在返回 `0`

### 本章会出现的 unordered_set 写法

```cpp
unordered_set<int> st(nums.begin(), nums.end());
```

这句话的意思是：

> 把 `nums` 数组里的所有数字都放进哈希集合 `st`。

例如：

```cpp
vector<int> nums = {100, 4, 200, 1, 3, 2};
unordered_set<int> st(nums.begin(), nums.end());
```

之后就可以快速判断某个数字是否存在：

```cpp
if (st.count(x - 1)) {
    // 说明 x - 1 存在
}
```

常见用途：

- 去重
- 判断是否存在
- 判断连续序列

### map / set

`map` 和 `set` 底层通常是红黑树，查找复杂度是 O(log n)，但 key 有序。

什么时候用？

- 需要按 key 有序遍历
- 需要找前驱/后继
- 不追求平均 O(1)

Hot100 哈希章节里，大多数情况优先用 `unordered_map` / `unordered_set`。

## 4. 哈希题常见模式

### 模式一：查找配对

代表题：两数之和

核心问题：

> 当前是 x，我想知道 target - x 之前有没有出现过。

### 模式二：按标准 key 分组

代表题：字母异位词分组

核心问题：

> 不同对象看起来不一样，但它们可以被转换成同一个标准形式。

例如：

- `eat`
- `tea`
- `ate`

排序后都变成 `aet`，所以可以分到一组。

### 模式三：快速判断存在性

代表题：最长连续序列

核心问题：

> 我想快速知道 x + 1、x - 1 是否存在。

### 模式四：计数

虽然本章 Hot100 题不完全展开，但后面很多题会用到：

```cpp
unordered_map<char, int> cnt;
for (char c : s) {
    cnt[c]++;
}
```

滑动窗口、字符串匹配、频率统计都会反复出现这个模式。

## 5. 面试中怎么判断要用哈希

看到这些关键词时，要立刻想到哈希：

- 是否出现过
- 出现次数
- 两个数凑目标值
- 分组
- 去重
- 查找补数
- 前缀和出现次数
- 未排序数组要求 O(n)

尤其是“未排序数组 + 要求 O(n)”时，哈希表经常是突破口。

# 二、题目 1：两数之和

链接：https://leetcode.cn/problems/two-sum/

标签：数组、哈希表  
难度：Easy  
重要程度：必须熟练到 5 分钟内写出

## 1. 题意简述

给定一个整数数组和目标值，找到两个不同位置的数，使它们相加等于目标值，返回这两个位置。

## 2. 这题考什么

这题不是考加法，而是考：

> 如何把“找另一个配对数字”这件事变快。

暴力法要枚举每一对数字。  
哈希表法只需要遍历一次，每次判断“我需要的那个数之前有没有出现过”。

## 3. 生活化解释

你和同学 AA 买东西，总价是 100 元。  
你手里看到一笔 37 元的付款记录，就会想：有没有另一笔 63 元？

笨方法是把所有付款记录两两配对。  
聪明方法是准备一个小本子：

- 看到 37 元时，先查本子里有没有 63 元
- 没有，就把 37 元记下来
- 后面看到 63 元时，查本子发现 37 元已经出现过，就找到答案

这个“小本子”就是 `unordered_map`。

## 4. 暴力思路

枚举所有 `i, j`：

下面这段是伪代码，不是 C++ 代码。  
伪代码的作用是表达思路，语法不要求能直接运行。

```text
for i in [0, n)
    for j in [i + 1, n)
        if nums[i] + nums[j] == target
            return {i, j}
```

对应的 C++ 代码是：

```cpp
for (int i = 0; i < nums.size(); i++) {
    for (int j = i + 1; j < nums.size(); j++) {
        if (nums[i] + nums[j] == target) {
            return {i, j};
        }
    }
}
```

复杂度：

- 时间复杂度：O(n^2)
- 空间复杂度：O(1)

缺点：数据大时太慢。

## 5. 哈希表思路

维护一个哈希表 `pos`：

- key：已经遍历过的数字
- value：这个数字的下标

遍历到 `nums[i]` 时：

1. 计算 `need = target - nums[i]`
2. 如果 `need` 已经出现过，返回 `{pos[need], i}`
3. 否则把 `nums[i]` 和下标 `i` 存起来

### 为什么要先查再放

因为题目要求不能使用同一个元素两次。

例如：

```text
nums = [3], target = 6
```

如果先把 3 放进去，再查 3，就会错误地把自己和自己配对。

所以顺序必须是：

```text
先查 need，再放当前 nums[i]
```

## 6. C++ 代码

```cpp
class Solution {
public:
    vector<int> twoSum(vector<int>& nums, int target) {
        unordered_map<int, int> pos;

        for (int i = 0; i < nums.size(); i++) {
            int need = target - nums[i];

            if (pos.count(need)) {
                return {pos[need], i};
            }

            pos[nums[i]] = i;
        }

        return {};
    }
};
```

## 7. 复杂度分析

- 时间复杂度：O(n)
- 空间复杂度：O(n)

`unordered_map` 查询平均 O(1)，所以整体平均 O(n)。

## 8. 易错点

1. 返回的是下标，不是数字本身。
2. 不能重复使用同一个元素。
3. 要先查 `need`，再插入当前数字。
4. 数组中可能有重复元素，哈希表存下标即可。

## 9. 面试表达

可以这样说：

> 暴力法需要两层循环，复杂度 O(n^2)。优化思路是用哈希表记录已经遍历过的数字及其下标。遍历当前数字 x 时，只需要判断 target - x 是否已经出现过。如果出现，就找到答案；否则记录当前数字。这样时间复杂度可以降到平均 O(n)。

## 10. 复刷要求

- 一刷：理解哈希查补数
- 二刷：3 分钟内独立写出
- 面试前：必须能一次 AC

# 三、题目 2：字母异位词分组

链接：https://leetcode.cn/problems/group-anagrams/

标签：哈希表、字符串、排序  
难度：Medium  
重要程度：哈希分组经典题

## 1. 题意简述

给定一组字符串，把由相同字母组成、只是排列顺序不同的字符串放到同一组。

例如：

```text
eat, tea, ate
```

这三个字符串都由 `a`、`e`、`t` 组成，所以应该分到同一组。

## 2. 这题考什么

这题考哈希表的分组能力：

> 如何为一类对象设计统一的 key。

如果两个字符串是字母异位词，它们一定有相同的字母组成。  
所以我们只要设计一个能表示“字母组成”的 key，就能分组。

## 3. 生活化解释

想象你在整理一堆拼字积木。

桌上有：

```text
eat, tea, tan, ate, nat, bat
```

这些单词看起来顺序不同，但如果你把每个单词的字母拆开并按字母表重新排好：

```text
eat -> aet
tea -> aet
ate -> aet
tan -> ant
nat -> ant
bat -> abt
```

排序后的结果就像一个“盒子标签”。  
标签一样的，放进同一个盒子。

## 4. 方法一：排序作为 key

对每个字符串：

1. 复制一份 `key`
2. 对 `key` 排序
3. 用排序后的 `key` 作为哈希表 key
4. 把原字符串放到对应数组中

例如：

```text
eat -> aet
tea -> aet
ate -> aet
```

它们会进入 `groups["aet"]`。

### 哈希表思路拆解

维护一个哈希表：

```cpp
unordered_map<string, vector<string>> groups;
```

它表示：

```text
标准 key -> 属于这一组的所有原字符串
```

在这道题里，标准 key 可以设计成“排序后的字符串”。

例如输入：

```text
["eat", "tea", "tan", "ate", "nat", "bat"]
```

处理过程是：

```text
处理 "eat"
key = "aet"
groups["aet"] = ["eat"]

处理 "tea"
key = "aet"
groups["aet"] = ["eat", "tea"]

处理 "tan"
key = "ant"
groups["ant"] = ["tan"]

处理 "ate"
key = "aet"
groups["aet"] = ["eat", "tea", "ate"]
```

所以核心不是“哈希表自动知道谁是一组”，而是我们人为设计了一个 key：

> 字母异位词排序后一定相同，所以用排序后的字符串当 key。

代码里的这句：

```cpp
groups[key].push_back(s);
```

意思是：

> 找到 `key` 对应的那一组，把原字符串 `s` 放进去。

如果 `key` 之前不存在，`groups[key]` 会自动创建一个空的 `vector<string>`，然后再把 `s` 放进去。

## 5. C++ 代码：排序法

```cpp
class Solution {
public:
    vector<vector<string>> groupAnagrams(vector<string>& strs) {
        unordered_map<string, vector<string>> groups;

        for (const string& s : strs) {
            string key = s;
            sort(key.begin(), key.end());
            groups[key].push_back(s);
        }

        vector<vector<string>> ans;
        for (auto& item : groups) {
            ans.push_back(item.second);
        }

        return ans;
    }
};
```

## 6. 复杂度分析：排序法

设：

- `n` 是字符串数量
- `k` 是字符串平均长度

每个字符串排序是 O(k log k)，一共有 n 个字符串。

- 时间复杂度：O(n * k log k)
- 空间复杂度：O(n * k)

## 7. 方法二：计数作为 key

如果字符串很长，排序成本会变高。  
因为题目通常是小写英文字母，所以可以统计每个字母出现次数。

例如：

```text
eat -> a:1, e:1, t:1
tea -> a:1, e:1, t:1
```

计数结果一样，就说明它们是字母异位词。

关键是要把计数数组转换成一个不会冲突的字符串 key。

错误示例：

```text
111
```

可能不知道是 `[1, 11]` 还是 `[11, 1]`。

正确做法是加分隔符：

```text
1#0#0#0#1#...#1#
```

### 计数法思路拆解

排序法的 key 是：

```text
把字符串排序后的结果
```

计数法的 key 是：

```text
每个字母出现次数组成的字符串
```

例如：

```text
eat
```

字母计数可以理解成：

```text
a 出现 1 次
b 出现 0 次
c 出现 0 次
...
e 出现 1 次
...
t 出现 1 次
```

于是生成 key：

```text
1#0#0#0#1#0#...#1#...
```

`tea` 和 `ate` 的计数结果完全一样，所以生成的 key 也一样，最后会被放进同一个 `groups[key]`。

代码里的：

```cpp
vector<int> cnt(26, 0);
```

表示创建一个长度为 26 的数组，分别记录 `a` 到 `z` 出现了几次。

代码里的：

```cpp
cnt[c - 'a']++;
```

表示把字符 `c` 映射到 `0-25` 的下标。

例如：

```text
'a' - 'a' = 0
'b' - 'a' = 1
'c' - 'a' = 2
```

所以 `cnt[0]` 记录 `a` 的次数，`cnt[1]` 记录 `b` 的次数。

## 8. C++ 代码：计数法

```cpp
class Solution {
public:
    vector<vector<string>> groupAnagrams(vector<string>& strs) {
        unordered_map<string, vector<string>> groups;

        for (const string& s : strs) {
            vector<int> cnt(26, 0);

            for (char c : s) {
                cnt[c - 'a']++;
            }

            string key;
            for (int x : cnt) {
                key += to_string(x);
                key += '#';
            }

            groups[key].push_back(s);
        }

        vector<vector<string>> ans;
        for (auto& item : groups) {
            ans.push_back(item.second);
        }

        return ans;
    }
};
```

## 9. 复杂度分析：计数法

- 时间复杂度：O(n * k)
- 空间复杂度：O(n * k)

计数法省去了排序，但 key 的构造略复杂。面试时如果没有特殊性能要求，排序法已经足够好。

## 10. 易错点

1. 不能直接用字符 ASCII 和作为 key。
   - `ad` 和 `bc` 字符和可能一样，但不是同一组。
2. 计数 key 要加分隔符。
3. 返回结果的顺序一般不要求固定。
4. `unordered_map<string, vector<string>>` 很适合这类分组题。

## 11. 面试表达

可以这样说：

> 这题的关键是给字母异位词构造相同的 key。最直观的方法是把每个字符串排序，排序后的字符串作为 key，用哈希表分组。这样相同字母组成的字符串会进入同一个 vector。也可以用 26 位计数数组作为 key，把时间复杂度从 O(n * k log k) 优化到 O(n * k)。

## 12. 复刷要求

- 一刷：掌握排序 key
- 二刷：能写出计数 key
- 面试前：能解释为什么不能用字符和做 key

# 四、题目 3：最长连续序列

链接：https://leetcode.cn/problems/longest-consecutive-sequence/

标签：数组、哈希集合  
难度：Medium  
重要程度：哈希集合经典题，常考

## 1. 题意简述

给一个未排序数组，找出最长的连续整数序列长度。

例如：

```text
100, 4, 200, 1, 3, 2
```

最长连续序列是：

```text
1, 2, 3, 4
```

长度是 4。

## 2. 这题考什么

这题考的是：

> 如何用哈希集合在未排序数组中快速判断一个数是否存在。

如果先排序，很容易做，但时间复杂度是 O(n log n)。  
题目通常希望你做到 O(n)，这时就要想到 `unordered_set`。

## 3. 生活化解释

你在整理电影票座位号，想找最长的一排连续座位。

手上有这些票：

```text
100, 4, 200, 1, 3, 2
```

你不应该从每张票都开始往后数。  
比如拿到 3 号票时，如果 2 号票也存在，那 3 号肯定不是这一排的开头。  
真正值得开始数的是 1 号，因为 0 号不存在，所以 1 号才是这段连续座位的起点。

这题的关键就是：

> 只从连续序列的起点开始扩展。

## 4. 方法一：排序

先排序，再扫描连续长度。

大致思路：

1. 排序数组
2. 跳过重复元素
3. 如果当前数等于前一个数 + 1，连续长度加一
4. 否则重新开始计数

这个方法好理解，但不是最优。

复杂度：

- 时间复杂度：O(n log n)
- 空间复杂度：取决于排序实现

## 5. 方法二：哈希集合

使用 `unordered_set` 存储所有数字。

然后遍历集合中的每个数字 `x`：

- 如果 `x - 1` 存在，说明 `x` 不是连续序列起点，跳过
- 如果 `x - 1` 不存在，说明 `x` 是起点
- 从 `x` 开始不断检查 `x + 1, x + 2, ...` 是否存在

这样每段连续序列只会从起点完整扫描一次。

### 哈希集合思路拆解

维护一个哈希集合：

```cpp
unordered_set<int> st(nums.begin(), nums.end());
```

它表示：

```text
数组里出现过哪些数字
```

和 `unordered_map` 不同，`unordered_set` 只记录“有没有”，不记录“对应什么值”。

例如：

```text
nums = [100, 4, 200, 1, 3, 2]
```

放入集合后，可以理解成：

```text
st = {100, 4, 200, 1, 3, 2}
```

接下来遍历每个数字 `x`。

关键判断是：

```cpp
if (st.count(x - 1)) {
    continue;
}
```

它的意思是：

> 如果 `x - 1` 存在，说明 `x` 前面还有数字，`x` 不是连续序列的起点，直接跳过。

例如：

```text
x = 3
x - 1 = 2
2 存在
```

说明 `3` 不是 `1,2,3,4` 这段序列的起点，所以不从 3 开始数。

只有当 `x - 1` 不存在时，才说明 `x` 是起点。

例如：

```text
x = 1
x - 1 = 0
0 不存在
```

所以从 1 开始向后扩展：

```text
1 存在
2 存在
3 存在
4 存在
5 不存在
```

这一段连续序列长度就是 4。

代码里的：

```cpp
while (st.count(cur + 1)) {
    cur++;
    len++;
}
```

意思是：

> 只要下一个连续数字存在，就继续往后数，并把当前长度加一。

## 6. 为什么是 O(n)

看起来有两层循环：

```cpp
for (int x : st) {
    while (st.count(cur + 1)) {
        cur++;
    }
}
```

但不是每个数字都会触发完整 `while`。

只有序列起点会触发向后扫描。  
例如 `1,2,3,4`：

- 从 1 开始扫描到 4
- 2、3、4 因为前一个数存在，直接跳过

所以所有 `while` 加起来最多扫描 O(n) 个元素。

## 7. C++ 代码：哈希集合

```cpp
class Solution {
public:
    int longestConsecutive(vector<int>& nums) {
        unordered_set<int> st(nums.begin(), nums.end());
        int ans = 0;

        for (int x : st) {
            if (st.count(x - 1)) {
                continue;
            }

            int cur = x;
            int len = 1;

            while (st.count(cur + 1)) {
                cur++;
                len++;
            }

            ans = max(ans, len);
        }

        return ans;
    }
};
```

## 8. 复杂度分析

- 时间复杂度：O(n)
- 空间复杂度：O(n)

`unordered_set` 的查找平均 O(1)。

## 9. 易错点

1. 空数组返回 0。
2. 要用 `unordered_set` 去重，否则重复数字会干扰。
3. 只从 `x - 1` 不存在的数字开始扩展。
4. 不要从每个数字都向后数，否则可能退化成 O(n^2)。

## 10. 面试表达

可以这样说：

> 排序后扫描可以做到 O(n log n)，但如果要求 O(n)，就用哈希集合。先把所有数字放入 set，然后只从连续序列的起点开始扩展。判断起点的方法是看 x - 1 是否存在。如果不存在，说明 x 是一段连续序列的开头，再不断检查 x + 1 是否存在。这样每个数字最多被扫描常数次，平均时间复杂度 O(n)。

## 11. 复刷要求

- 一刷：理解为什么只从起点扩展
- 二刷：能解释 O(n) 的原因
- 面试前：能和排序法做对比

# 五、哈希章节总结

## 1. 三道题对应三种哈希能力

| 题目 | 哈希能力 | 关键词 |
|---|---|---|
| 两数之和 | 快速查找配对元素 | `target - x` |
| 字母异位词分组 | 构造标准 key 分组 | 排序 key / 计数 key |
| 最长连续序列 | 快速判断元素存在 | 只从序列起点扩展 |

## 2. C++ 模板回顾

### 查找配对

```cpp
unordered_map<int, int> mp;
for (int i = 0; i < nums.size(); i++) {
    int need = target - nums[i];
    if (mp.count(need)) {
        return {mp[need], i};
    }
    mp[nums[i]] = i;
}
```

### 分组

```cpp
unordered_map<string, vector<string>> groups;
for (const string& s : strs) {
    string key = s;
    sort(key.begin(), key.end());
    groups[key].push_back(s);
}
```

### 快速判断存在

```cpp
unordered_set<int> st(nums.begin(), nums.end());
if (st.count(x)) {
    // x exists
}
```

## 3. 哈希表使用注意事项

1. `unordered_map` 是平均 O(1)，不是严格 O(1)。
2. 需要有序时用 `map`，否则优先 `unordered_map`。
3. 分组题的 key 必须唯一、稳定、不会冲突。
4. 计数数组转字符串时要加分隔符。
5. 面试时要能说清楚为什么用哈希，而不是只说“用 map”。
