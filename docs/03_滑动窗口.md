# LeetCode Hot 100 - 03 滑动窗口 C++ 题解

## 一、滑动窗口知识点回顾

滑动窗口专门处理“连续区间”问题，可以是字符串子串，也可以是数组子数组。它像一个会伸缩的取景框：右边界负责把新元素纳入窗口，左边界负责在窗口不合法或已经满足条件时收缩。

常见判断：

- 固定窗口：窗口长度固定，例如找所有异位词。
- 可变窗口：窗口长度不固定，例如最长无重复子串、最小覆盖子串。
- 伪滑窗题：看似连续区间，但因为有负数导致窗口和没有单调性，例如“和为 K 的子数组”，要改用前缀和。

常用 C++ 容器：

```cpp
unordered_map<char, int> cnt;      // 字符计数
vector<int> freq(26, 0);           // 小写字母频率
deque<int> q;                      // 单调队列
```

`deque` 是双端队列：

- `push_back(x)`：队尾加入
- `pop_back()`：弹出队尾
- `pop_front()`：弹出队头
- `front()`：访问队头
- `back()`：访问队尾

## 1. 无重复字符的最长子串

链接：https://leetcode.cn/problems/longest-substring-without-repeating-characters/

标签：字符串、哈希表、滑动窗口  
难度：Medium  
重要程度：滑动窗口入门必会

### 题意简述

给定一个字符串，找出其中不含重复字符的最长连续子串长度。

### 这题考什么

考察可变滑动窗口，以及如何用哈希表维护窗口内字符出现次数。

### 生活化理解

像一个房间里不能有两个同名的人。右边不断进人，一旦来了重名的人，就从左边开始请人出去，直到房间里重新没有重名。

### 解题思路

维护窗口 `[left, right]`。右指针每次加入一个字符并增加计数；如果当前字符出现次数超过 1，说明窗口不合法，就移动左指针并减少左侧字符计数，直到窗口重新合法。每次窗口合法后，用 `right - left + 1` 更新答案。

### C++ 代码

```cpp
class Solution {
public:
    int lengthOfLongestSubstring(string s) {
        unordered_map<char, int> cnt;
        int left = 0;
        int ans = 0;

        for (int right = 0; right < s.size(); right++) {
            cnt[s[right]]++;

            while (cnt[s[right]] > 1) {
                cnt[s[left]]--;
                left++;
            }

            ans = max(ans, right - left + 1);
        }

        return ans;
    }
};
```

### 复杂度分析

- 时间复杂度：O(n)，每个字符最多进出窗口一次。
- 空间复杂度：O(字符集大小)。

### 易错点

- 窗口长度是 `right - left + 1`。
- 收缩窗口时减少的是 `s[left]` 的计数。
- 不要只用 set 删除字符，重复字符次数在某些写法中很重要。

### 面试表达

用滑动窗口维护一个无重复字符区间。右指针扩张，出现重复后左指针收缩，直到窗口重新合法。整个过程每个字符最多被左右指针各访问一次。

### 复刷要求

- 一刷理解可变窗口。
- 二刷能 5 分钟内写出。
- 面试前能解释为什么是 O(n)。

## 2. 找到字符串中所有字母异位词

链接：https://leetcode.cn/problems/find-all-anagrams-in-a-string/

标签：字符串、滑动窗口、频率数组  
难度：Medium  
重要程度：固定窗口经典题

### 题意简述

在字符串 `s` 中找到所有长度等于 `p`，且字符组成和 `p` 相同的子串起始下标。

### 这题考什么

考察固定长度滑动窗口，以及用频率数组判断两个字符串是否为字母异位词。

### 生活化理解

像拿一把长度固定的尺子在长字符串上滑动。尺子盖住的每一段都拿来和目标单词比对，看 26 个字母数量是否完全相同。

### 解题思路

窗口长度固定为 `p.size()`。用 `need` 记录 `p` 的字母频率，用 `window` 记录当前窗口频率。右指针加入字符；当窗口长度超过 `p.size()`，左指针移出字符；当窗口长度刚好等于 `p.size()` 且两个频率数组相等，记录左端点。

### C++ 代码

```cpp
class Solution {
public:
    vector<int> findAnagrams(string s, string p) {
        vector<int> ans;
        if (s.size() < p.size()) return ans;

        vector<int> need(26, 0), window(26, 0);
        for (char c : p) need[c - 'a']++;

        int left = 0;
        for (int right = 0; right < s.size(); right++) {
            window[s[right] - 'a']++;

            if (right - left + 1 > p.size()) {
                window[s[left] - 'a']--;
                left++;
            }

            if (right - left + 1 == p.size() && window == need) {
                ans.push_back(left);
            }
        }

        return ans;
    }
};
```

### 复杂度分析

- 时间复杂度：O(26n)，可视为 O(n)。
- 空间复杂度：O(1)，频率数组大小固定为 26。

### 易错点

- 窗口长度超过目标长度时要先移除左边。
- `window == need` 是比较 26 个计数，不是比较排序后的字符串。
- 如果 `s.size() < p.size()`，直接返回空数组。

### 面试表达

这是固定窗口题。窗口大小等于 p 的长度，滑动过程中维护当前窗口的 26 位字符计数，计数与目标字符串一致时记录起点。

### 复刷要求

- 一刷掌握固定窗口。
- 二刷尝试用 valid 计数优化比较。
- 面试前能区分它和最小覆盖子串。

## 3. 和为 K 的子数组

链接：https://leetcode.cn/problems/subarray-sum-equals-k/

标签：数组、前缀和、哈希表  
难度：Medium  
重要程度：前缀和高频题

### 题意简述

给定整数数组和整数 `k`，统计和为 `k` 的连续子数组数量。

### 这题考什么

这题看似滑动窗口，但因为数组可能有负数，窗口和不具备单调性，所以考察前缀和 + 哈希表。

### 生活化理解

像记流水账。当前位置累计消费是 `sum`，如果之前有某个时间点累计消费是 `sum - k`，那么这两个时间点之间的消费正好是 `k`。

### 解题思路

维护当前前缀和 `sum`，以及哈希表 `cnt` 记录某个前缀和出现过多少次。遍历到当前元素后，如果之前出现过 `sum - k`，这些位置都能和当前位置组成和为 k 的子数组。初始化 `cnt[0] = 1`，表示空前缀出现一次。

### C++ 代码

```cpp
class Solution {
public:
    int subarraySum(vector<int>& nums, int k) {
        unordered_map<int, int> cnt;
        cnt[0] = 1;

        int sum = 0;
        int ans = 0;

        for (int x : nums) {
            sum += x;
            if (cnt.count(sum - k)) {
                ans += cnt[sum - k];
            }
            cnt[sum]++;
        }

        return ans;
    }
};
```

### 复杂度分析

- 时间复杂度：O(n)。
- 空间复杂度：O(n)。

### 易错点

- 不能用普通滑动窗口，因为有负数。
- `cnt[0] = 1` 不能忘，否则漏掉从下标 0 开始的子数组。
- 必须先统计答案，再把当前前缀和加入哈希表。

### 面试表达

用前缀和把子数组和转成两个前缀和之差。当前前缀和为 sum 时，之前每个 sum-k 都对应一个和为 k 的子数组。

### 复刷要求

- 一刷理解为什么不是滑动窗口。
- 二刷能独立写出 `cnt[0] = 1`。
- 面试前能讲出先查后加的原因。

## 4. 滑动窗口最大值

链接：https://leetcode.cn/problems/sliding-window-maximum/

标签：数组、队列、单调队列  
难度：Hard  
重要程度：单调队列经典题

### 题意简述

给定数组和窗口大小 `k`，窗口每次向右滑动一格，返回每个窗口中的最大值。

### 这题考什么

考察单调队列。要动态维护窗口最大值，同时删除过期元素。

### 生活化理解

像一个候选人队伍，每次只保留可能成为当前或未来窗口最大值的人。新来的人如果比队尾候选人更强，队尾就不可能再当最大值，可以删掉。

### 解题思路

用 `deque<int>` 存下标，并让队列对应的值保持从大到小。每处理一个下标 `i`：先弹出窗口外的队头；再弹出所有比 `nums[i]` 小或等于的队尾；最后把 `i` 入队。窗口形成后，队头就是最大值下标。

### C++ 代码

```cpp
class Solution {
public:
    vector<int> maxSlidingWindow(vector<int>& nums, int k) {
        vector<int> ans;
        deque<int> q;

        for (int i = 0; i < nums.size(); i++) {
            while (!q.empty() && q.front() <= i - k) {
                q.pop_front();
            }

            while (!q.empty() && nums[q.back()] <= nums[i]) {
                q.pop_back();
            }

            q.push_back(i);

            if (i >= k - 1) {
                ans.push_back(nums[q.front()]);
            }
        }

        return ans;
    }
};
```

### 复杂度分析

- 时间复杂度：O(n)，每个下标最多进队出队一次。
- 空间复杂度：O(k)。

### 易错点

- 队列里存下标，不要只存值。
- 判断过期用 `q.front() <= i - k`。
- 窗口形成条件是 `i >= k - 1`。

### 面试表达

用单调递减队列维护最大值候选，队头始终是当前窗口最大值。新元素入队前移除过期下标和所有不可能再成为最大值的队尾元素。

### 复刷要求

- 一刷理解单调队列。
- 二刷能解释为什么每个元素最多进出一次。
- 面试前能手写 deque 操作。

## 5. 最小覆盖子串

链接：https://leetcode.cn/problems/minimum-window-substring/

标签：字符串、哈希表、滑动窗口  
难度：Hard  
重要程度：可变窗口压轴题

### 题意简述

在字符串 `s` 中找到最短子串，使它包含字符串 `t` 中所有字符及其数量。

### 这题考什么

考察可变窗口、字符计数、满足条件后尽量收缩窗口。

### 生活化理解

像在一排货架里找最短的一段货架，里面必须包含购物清单上的所有商品。右边不断扩张直到商品齐了，然后左边尽量往右缩短。

### 解题思路

用 `need` 记录目标字符需求，用 `window` 记录当前窗口字符数量。`valid` 表示有多少种字符已经满足要求。当 `valid == need.size()`，说明窗口覆盖了 `t`，此时更新答案并收缩左边界；收缩导致某字符不再满足时，`valid--`。

### C++ 代码

```cpp
class Solution {
public:
    string minWindow(string s, string t) {
        unordered_map<char, int> need, window;
        for (char c : t) need[c]++;

        int left = 0;
        int valid = 0;
        int start = 0;
        int len = INT_MAX;

        for (int right = 0; right < s.size(); right++) {
            char c = s[right];
            if (need.count(c)) {
                window[c]++;
                if (window[c] == need[c]) valid++;
            }

            while (valid == need.size()) {
                if (right - left + 1 < len) {
                    start = left;
                    len = right - left + 1;
                }

                char d = s[left];
                left++;
                if (need.count(d)) {
                    if (window[d] == need[d]) valid--;
                    window[d]--;
                }
            }
        }

        return len == INT_MAX ? "" : s.substr(start, len);
    }
};
```

### 复杂度分析

- 时间复杂度：O(n + m)。
- 空间复杂度：O(字符集大小)。

### 易错点

- `valid` 统计的是满足需求的字符种类数，不是字符总数。
- 收缩时如果当前字符刚好满足需求，要先 `valid--` 再减少计数。
- 答案为空时返回空字符串。

### 面试表达

右指针扩张窗口，直到窗口覆盖 t；覆盖后左指针尽量收缩，并更新最短答案。用 need/window 两个哈希表维护字符数量，用 valid 判断是否满足所有字符种类。

### 复刷要求

- 一刷理解 need/window/valid。
- 二刷能独立处理收缩逻辑。
- 面试前能和固定窗口异位词题对比。

## 本章总结

本章核心是先判断题目是不是连续区间，再判断窗口固定还是可变。固定窗口看长度，可变窗口看合法性。遇到负数和问题时要警惕，普通滑动窗口可能失效。
