## 一、知识点回顾

滑动窗口解决的是“连续区间”问题。这个连续区间可以是字符串里的子串，也可以是数组里的子数组。

你可以把滑动窗口想象成一个会伸缩的取景框：

- 右指针 `right` 负责把新元素纳入窗口。
- 左指针 `left` 负责把旧元素移出窗口。
- 窗口内部维护某种状态，比如字符出现次数、当前窗口长度、当前最大值候选。

滑动窗口常见三类：

| 类型 | 特征 | 代表题 |
|---|---|---|
| 可变窗口 | 窗口长度不固定，满足条件后收缩或不满足时收缩 | 无重复字符的最长子串、最小覆盖子串 |
| 固定窗口 | 窗口长度固定，每次滑动一格 | 找到所有字母异位词 |
| 单调队列窗口 | 窗口固定，但要动态维护最大/最小值 | 滑动窗口最大值 |

还有一种“看起来像滑动窗口但不能直接用”的题：

| 类型 | 为什么不能直接滑窗 | 代表题 |
|---|---|---|
| 前缀和 + 哈希 | 数组中有负数，窗口和不具备单调性 | 和为 K 的子数组 |

普通滑动窗口依赖一种隐含前提：当窗口扩大或缩小时，某个状态有可控变化。  
例如全是正数时，窗口变大总和会变大，窗口缩小总和会变小。  
但如果数组里有负数，窗口变大可能反而变小，所以不能简单用“和大了左移、和小了右移”。

## 二、基础容器/语法解释

本章常用 C++ 容器：

```cpp
unordered_map<char, int> cnt;
vector<int> freq(26, 0);
deque<int> q;
```

### 1. unordered_map<char, int>

用于统计字符出现次数。

```cpp
unordered_map<char, int> cnt;
cnt['a']++;
cnt['a']--;
cnt.count('a');
```

含义：

- `cnt['a']++`：字符 `'a'` 出现次数加一。
- `cnt['a']--`：字符 `'a'` 出现次数减一。
- `cnt.count('a')`：判断 `'a'` 这个 key 是否存在。

注意：

```cpp
cnt[c]
```

如果 `c` 不存在，会自动创建这个 key，默认 value 是 0。  
所以只是判断存在时，优先用 `count()`。

### 2. vector<int> freq(26, 0)

用于统计 26 个小写字母出现次数。

```cpp
vector<int> freq(26, 0);
freq[c - 'a']++;
```

`c - 'a'` 可以把小写字母映射为 `0-25`：

```text
'a' - 'a' = 0
'b' - 'a' = 1
'z' - 'a' = 25
```

所以：

- `freq[0]` 表示 `a` 的次数。
- `freq[1]` 表示 `b` 的次数。
- `freq[25]` 表示 `z` 的次数。

### 3. deque<int>

`deque` 是双端队列，两头都能进出。

```cpp
deque<int> q;
q.push_back(i);
q.pop_back();
q.pop_front();
q.front();
q.back();
```

含义：

- `push_back(i)`：从队尾加入元素。
- `pop_back()`：删除队尾元素。
- `pop_front()`：删除队头元素。
- `front()`：访问队头元素。
- `back()`：访问队尾元素。

滑动窗口最大值里，`deque` 里通常存的是“下标”，不是值。  
原因是：我们既要比较值大小，也要判断某个元素是否已经滑出窗口。

## 三、滑动窗口通用模板

### 可变窗口模板

```cpp
int left = 0;
for (int right = 0; right < n; right++) {
    // 1. 把 nums[right] 或 s[right] 加入窗口

    while (窗口不满足条件) {
        // 2. 移除 nums[left] 或 s[left]
        left++;
    }

    // 3. 在窗口合法时更新答案
}
```

适合：

- 最长无重复子串
- 最小覆盖子串

### 固定窗口模板

```cpp
int left = 0;
for (int right = 0; right < n; right++) {
    // 1. 加入右边元素

    if (right - left + 1 > k) {
        // 2. 移除左边元素
        left++;
    }

    if (right - left + 1 == k) {
        // 3. 更新答案
    }
}
```

适合：

- 找到所有字母异位词

### 单调队列模板

```cpp
deque<int> q;
for (int i = 0; i < nums.size(); i++) {
    while (!q.empty() && q.front() <= i - k) {
        q.pop_front();
    }

    while (!q.empty() && nums[q.back()] <= nums[i]) {
        q.pop_back();
    }

    q.push_back(i);

    if (i >= k - 1) {
        ans.push_back(nums[q.front()]);
    }
}
```

适合：

- 滑动窗口最大值

## 1. 无重复字符的最长子串

链接：https://leetcode.cn/problems/longest-substring-without-repeating-characters/

标签：字符串、哈希表、滑动窗口  
难度：Medium  
重要程度：滑动窗口入门必会

### 题意简述

给定一个字符串，找出其中不含重复字符的最长连续子串长度。

注意关键词：

- 子串：必须连续。
- 无重复字符：窗口内每个字符最多出现一次。
- 返回长度，不是返回子串本身。

### 这题考什么

这题考察可变滑动窗口：

> 右指针不断扩张窗口；一旦窗口里出现重复字符，就移动左指针收缩窗口，直到窗口重新合法。

它是滑动窗口最典型的入门题，因为窗口合法条件非常直观：

```text
窗口内没有重复字符
```

### 生活化理解

想象一个小房间，规则是：房间里不能有两个同名的人。

现在大家从右边排队进房间：

- 来了一个新名字，没重复，就让他进来。
- 如果来了一个和房间里某人同名的人，房间就违规了。
- 这时从左边开始请人出去，直到房间里不再有重名。

每一次房间合法时，记录房间里最多能容纳多少人。

这个房间就是滑动窗口。

### 核心思路

先用直观方法理解题目，再提炼窗口状态、移动规则和答案更新时机。下面通过方法一和方法二对比，明确为什么推荐优化解法。

### 方法一：暴力枚举

最直接的方法是枚举每个起点，再往右扩展，直到遇到重复字符。

伪代码：

```text
for 每个起点 i:
    清空 seen
    for 每个终点 j:
        如果 s[j] 已出现，停止
        否则加入 seen，更新答案
```

复杂度：

- 时间复杂度：O(n^2)
- 空间复杂度：O(字符集大小)

这个方法能过小数据，但面试中不是推荐答案。

### 方法二：滑动窗口 + 计数表

维护：

```cpp
unordered_map<char, int> cnt;
int left = 0;
```

含义：

- `cnt`：记录当前窗口中每个字符出现次数。
- `left`：窗口左边界。
- `right`：窗口右边界，由 for 循环控制。

每次加入 `s[right]`：

```cpp
cnt[s[right]]++;
```

如果 `s[right]` 出现次数超过 1，说明窗口不合法：

```cpp
while (cnt[s[right]] > 1) {
    cnt[s[left]]--;
    left++;
}
```

窗口恢复合法后，更新答案：

```cpp
ans = max(ans, right - left + 1);
```

### C++ 题解

```cpp
class Solution {
public:
    int lengthOfLongestSubstring(string s) {
        unordered_map<char, int> cnt;
        int left = 0;
        int ans = 0;

        for (int right = 0; right < s.size(); right++) {
            cnt[s[right]]++;

            while (cnt[s[right]] > 1) {
                cnt[s[left]]--;
                left++;
            }

            ans = max(ans, right - left + 1);
        }

        return ans;
    }
};
```

### 复杂度分析

- 时间复杂度：O(n)
  - `right` 从左到右走一遍。
  - `left` 也最多从左到右走一遍。
- 空间复杂度：O(字符集大小)
  - 如果只考虑 ASCII，可以看作 O(1)。
  - 如果字符集很大，则与不同字符数量有关。

### 易错点

- 窗口长度是 `right - left + 1`。
- 收缩窗口时减少的是 `s[left]` 的计数。
- `while` 条件只需要看当前加入的字符 `s[right]` 是否重复。
- 题目要的是“子串”，不是“子序列”。

### 面试表达

可以这样说：

> 我用滑动窗口维护一个没有重复字符的区间。右指针每次加入一个字符，如果该字符出现次数超过 1，说明窗口不合法，就移动左指针并减少左侧字符计数，直到窗口重新合法。每次合法后更新最大长度。由于左右指针都只单向移动，所以时间复杂度是 O(n)。

### 复刷要求

- 一刷：理解为什么出现重复后要移动左指针。
- 二刷：能不看题解写出 `while (cnt[s[right]] > 1)`。
- 面试前：能手推 `abcabcbb`。

## 2. 找到字符串中所有字母异位词

链接：https://leetcode.cn/problems/find-all-anagrams-in-a-string/

标签：字符串、滑动窗口、频率数组  
难度：Medium  
重要程度：固定窗口经典题

### 题意简述

给定字符串 `s` 和 `p`，找到 `s` 中所有是 `p` 的字母异位词的子串，返回这些子串的起始下标。

字母异位词的意思是：

```text
字母种类相同，字母数量相同，只是顺序不同
```

例如：

```text
abc、bac、cba
```

都是同一组异位词。

### 这题考什么

这题考固定长度滑动窗口：

> 因为异位词长度一定等于 `p.size()`，所以窗口大小固定。

同时考察频率数组：

> 只要窗口内 26 个字母出现次数和 `p` 完全相同，就说明当前窗口是一个异位词。

### 生活化理解

你拿着一个长度固定的透明尺子，在字符串 `s` 上从左往右滑。

尺子的长度等于 `p` 的长度。  
每滑到一个位置，就看尺子盖住的这段字符：

- `a` 有几个？
- `b` 有几个？
- ...
- `z` 有几个？

如果 26 个字母数量和 `p` 完全一样，就记录尺子左端的位置。

### 核心思路

先用直观方法理解题目，再提炼窗口状态、移动规则和答案更新时机。下面通过方法一和方法二对比，明确为什么推荐优化解法。

### 方法一：每个窗口排序

对每个长度为 `p.size()` 的窗口排序，再和排序后的 `p` 比较。

复杂度：

- 每个窗口排序：O(k log k)
- 总窗口数约 n
- 总时间：O(n * k log k)

这个方法直观，但不够优。

### 方法二：固定窗口 + 频率数组

维护两个数组：

```cpp
vector<int> need(26, 0);
vector<int> window(26, 0);
```

含义：

- `need`：目标字符串 `p` 的字母频率。
- `window`：当前窗口的字母频率。

流程：

1. 先统计 `p` 的字母频率。
2. 右指针加入新字符。
3. 如果窗口长度超过 `p.size()`，左指针移出旧字符。
4. 当窗口长度刚好等于 `p.size()`，比较 `window == need`。

### C++ 题解

```cpp
class Solution {
public:
    vector<int> findAnagrams(string s, string p) {
        vector<int> ans;
        if (s.size() < p.size()) return ans;

        vector<int> need(26, 0), window(26, 0);
        for (char c : p) {
            need[c - 'a']++;
        }

        int left = 0;
        for (int right = 0; right < s.size(); right++) {
            window[s[right] - 'a']++;

            if (right - left + 1 > p.size()) {
                window[s[left] - 'a']--;
                left++;
            }

            if (right - left + 1 == p.size() && window == need) {
                ans.push_back(left);
            }
        }

        return ans;
    }
};
```

### 复杂度分析

- 时间复杂度：O(26n)，可视为 O(n)
  - 每次比较两个长度为 26 的数组。
- 空间复杂度：O(1)
  - 两个长度为 26 的数组。

### 易错点

- 如果 `s.size() < p.size()`，直接返回空数组。
- 窗口长度超过 `p.size()` 时，要先移除左边字符。
- `window == need` 比较的是 26 个频率，不是字符串顺序。
- 记录的是起始下标 `left`。

### 面试表达

可以这样说：

> 因为异位词长度固定等于 p 的长度，所以我使用固定大小滑动窗口。用两个 26 位数组分别记录目标字符串和当前窗口的字符频率。窗口向右滑动时加入右端字符，窗口过长时移除左端字符。当窗口长度等于 p 长度且频率数组相等时，当前左端点就是答案。

### 复刷要求

- 一刷：掌握固定窗口写法。
- 二刷：能解释为什么数组比较是 O(26)。
- 面试前：能和“最小覆盖子串”的可变窗口区分开。

## 3. 和为 K 的子数组

链接：https://leetcode.cn/problems/subarray-sum-equals-k/

标签：数组、前缀和、哈希表  
难度：Medium  
重要程度：前缀和高频题

### 题意简述

给定整数数组 `nums` 和整数 `k`，统计和为 `k` 的连续子数组数量。

关键词：

- 连续子数组。
- 统计数量，不是找最长或最短。
- 数组中可能有负数。

### 这题考什么

这题放在滑动窗口章节里很容易误导。  
它真正考的是：

> 前缀和 + 哈希表。

为什么不能直接滑动窗口？

因为数组里可能有负数。  
当右指针右移时，窗口和不一定变大；当左指针右移时，窗口和也不一定变小。

例如：

```text
[1, -1, 1]
```

窗口加入 `-1` 后，总和反而下降。  
所以普通“和大了左移，和小了右移”的策略会失效。

### 生活化理解

你在记流水账。

从第一天到今天的累计收入是 `sum`。  
如果你想知道有没有一段时间收入正好是 `k`，只要看以前有没有某天的累计收入是：

```text
sum - k
```

因为：

```text
今天累计收入 - 以前某天累计收入 = 中间这段收入
```

如果以前有多个这样的累计收入，每一个都对应一段和为 `k` 的子数组。

### 核心思路

先用直观方法理解题目，再提炼窗口状态、移动规则和答案更新时机。下面通过方法一和方法二对比，明确为什么推荐优化解法。

### 方法一：暴力枚举

枚举每个起点和终点，计算区间和。

优化一点可以边枚举终点边累加，避免重复求和：

```text
for i in [0, n):
    sum = 0
    for j in [i, n):
        sum += nums[j]
        if sum == k: ans++
```

复杂度：

- 时间复杂度：O(n^2)
- 空间复杂度：O(1)

### 方法二：前缀和 + 哈希表

定义前缀和：

```text
prefix[i] = nums[0] + nums[1] + ... + nums[i]
```

如果某段子数组和为 `k`，说明：

```text
prefix[right] - prefix[left - 1] = k
```

变形：

```text
prefix[left - 1] = prefix[right] - k
```

所以遍历到当前前缀和 `sum` 时，只要知道之前有多少个 `sum - k`，就知道以当前位置结尾的和为 `k` 的子数组有多少个。

哈希表：

```cpp
unordered_map<int, int> cnt;
```

含义：

```text
某个前缀和 -> 出现次数
```

初始化：

```cpp
cnt[0] = 1;
```

表示空前缀出现过一次。  
这一步非常关键，用来处理从下标 0 开始的子数组。

### C++ 题解

```cpp
class Solution {
public:
    int subarraySum(vector<int>& nums, int k) {
        unordered_map<int, int> cnt;
        cnt[0] = 1;

        int sum = 0;
        int ans = 0;

        for (int x : nums) {
            sum += x;

            if (cnt.count(sum - k)) {
                ans += cnt[sum - k];
            }

            cnt[sum]++;
        }

        return ans;
    }
};
```

### 复杂度分析

- 时间复杂度：O(n)
  - 遍历数组一次，哈希表查询平均 O(1)。
- 空间复杂度：O(n)
  - 最坏情况下每个前缀和都不同。

### 易错点

- 不能用普通滑动窗口。
- `cnt[0] = 1` 不能忘。
- 必须先查询 `sum - k`，再记录当前 `sum`。
- 统计的是数量，所以要加 `cnt[sum - k]`，不是只加 1。

### 面试表达

可以这样说：

> 由于数组中可能有负数，窗口和不具备单调性，所以不能用普通滑动窗口。我使用前缀和加哈希表。当前前缀和为 sum 时，如果之前出现过 sum - k，那么这些位置到当前位置之间的子数组和就是 k。哈希表记录每个前缀和出现次数，遍历时累加答案。

### 复刷要求

- 一刷：理解为什么不能滑窗。
- 二刷：能写出 `cnt[0] = 1`。
- 面试前：能手推 `[1,1,1], k=2`。

## 4. 滑动窗口最大值

链接：https://leetcode.cn/problems/sliding-window-maximum/

标签：数组、队列、单调队列、滑动窗口  
难度：Hard  
重要程度：单调队列经典题

### 题意简述

给定数组 `nums` 和窗口大小 `k`，窗口每次向右滑动一格，返回每个窗口中的最大值。

例如：

```text
窗口 [1,3,-1] 的最大值是 3
窗口 [3,-1,-3] 的最大值是 3
```

### 这题考什么

这题考察单调队列：

> 如何在窗口滑动时快速得到当前窗口最大值。

如果每个窗口都重新扫描找最大值，时间复杂度是 O(nk)。  
单调队列可以把它优化到 O(n)。

### 生活化理解

想象一个候选人队伍，窗口里要选最高的人。

如果新来了一个更高的人，那么他前面那些比他矮、且更早进入窗口的人，就不可能在未来成为最大值了。  
因为：

- 新来的人更高。
- 新来的人更晚离开窗口。

所以这些矮个候选人可以直接从队尾淘汰。

队头始终保留当前窗口最强候选。

### 核心思路

先用直观方法理解题目，再提炼窗口状态、移动规则和答案更新时机。下面通过方法一和方法二对比，明确为什么推荐优化解法。

### 方法一：暴力扫描

每个窗口都扫描 k 个元素找最大值。

复杂度：

- 时间复杂度：O(nk)
- 空间复杂度：O(1)

当 `k` 很大时会超时。

### 方法二：单调队列

用 `deque<int>` 存下标，并保持队列中下标对应的值单调递减。

为什么存下标？

1. 可以通过 `nums[index]` 比较值。
2. 可以通过下标判断是否已经滑出窗口。

每次处理下标 `i`：

1. 删除过期下标：

```cpp
while (!q.empty() && q.front() <= i - k) q.pop_front();
```

2. 删除队尾中小于等于当前值的下标：

```cpp
while (!q.empty() && nums[q.back()] <= nums[i]) q.pop_back();
```

3. 当前下标入队：

```cpp
q.push_back(i);
```

4. 如果窗口已经形成，队头就是最大值：

```cpp
if (i >= k - 1) ans.push_back(nums[q.front()]);
```

### C++ 题解

```cpp
class Solution {
public:
    vector<int> maxSlidingWindow(vector<int>& nums, int k) {
        vector<int> ans;
        deque<int> q;

        for (int i = 0; i < nums.size(); i++) {
            while (!q.empty() && q.front() <= i - k) {
                q.pop_front();
            }

            while (!q.empty() && nums[q.back()] <= nums[i]) {
                q.pop_back();
            }

            q.push_back(i);

            if (i >= k - 1) {
                ans.push_back(nums[q.front()]);
            }
        }

        return ans;
    }
};
```

### 复杂度分析

- 时间复杂度：O(n)
  - 每个下标最多入队一次、出队一次。
- 空间复杂度：O(k)
  - 队列最多保存一个窗口范围内的候选下标。

### 易错点

- 队列存下标，不是存值。
- 判断过期是 `q.front() <= i - k`。
- 窗口形成条件是 `i >= k - 1`。
- 队列保持的是从大到小，不是从小到大。

### 面试表达

可以这样说：

> 我用单调递减队列维护窗口最大值候选。队列中存下标，队头始终是当前窗口最大值下标。遍历每个位置时，先移除过期下标，再从队尾删除所有小于等于当前元素的下标，因为它们不可能再成为最大值。窗口形成后，队头对应的值就是答案。

### 复刷要求

- 一刷：理解为什么小的队尾可以删。
- 二刷：能写出过期判断。
- 面试前：能手推 `nums=[1,3,-1,-3,5,3,6,7], k=3`。

## 5. 最小覆盖子串

链接：https://leetcode.cn/problems/minimum-window-substring/

标签：字符串、哈希表、滑动窗口  
难度：Hard  
重要程度：可变窗口压轴题

### 题意简述

给定字符串 `s` 和 `t`，在 `s` 中找到最短子串，使它包含 `t` 中所有字符及其数量。

注意：

- 必须包含 `t` 的所有字符。
- 字符数量也要满足。
- 返回最短的那个子串。

### 这题考什么

这题考察可变滑动窗口的完整形态：

> 右指针扩张直到窗口满足条件；满足条件后，左指针尽量收缩，寻找最短答案。

它比“无重复字符的最长子串”更复杂，因为窗口合法条件不是“没有重复”，而是：

```text
窗口中每个目标字符的数量都达到 need 要求
```

### 生活化理解

你去超市买东西，购物清单是 `t`。

你从货架左到右框出一段区域：

- 右边界不断向右扩张，直到这段区域包含清单上所有商品。
- 一旦商品齐了，就尝试移动左边界，把不必要的商品排除掉。
- 每次商品仍然齐全时，都记录当前更短的区域。

最后找到最短的一段货架。

### 核心思路

先用直观方法理解题目，再提炼窗口状态、移动规则和答案更新时机。下面通过方法一和方法二对比，明确为什么推荐优化解法。

### 方法一：暴力枚举

枚举所有子串，检查是否覆盖 `t`。

复杂度很高：

- 子串数量 O(n^2)
- 每次检查字符频率

面试中不推荐。

### 方法二：滑动窗口 + need/window

维护两个哈希表：

```cpp
unordered_map<char, int> need;
unordered_map<char, int> window;
```

含义：

- `need`：目标字符串中每个字符需要多少个。
- `window`：当前窗口中每个目标字符已有多少个。

再维护：

```cpp
int valid = 0;
```

含义：

> 当前窗口中，有多少种字符已经满足 need 的数量要求。

当：

```cpp
valid == need.size()
```

说明窗口已经覆盖 `t`。

流程：

1. 右指针加入字符。
2. 如果这个字符属于 `need`，更新 `window`。
3. 当某字符数量刚好满足要求时，`valid++`。
4. 当窗口满足要求时，更新最短答案，并移动左指针收缩。
5. 如果移出的字符导致窗口不再满足要求，`valid--`。

### C++ 题解

```cpp
class Solution {
public:
    string minWindow(string s, string t) {
        unordered_map<char, int> need, window;
        for (char c : t) {
            need[c]++;
        }

        int left = 0;
        int valid = 0;
        int start = 0;
        int len = INT_MAX;

        for (int right = 0; right < s.size(); right++) {
            char c = s[right];

            if (need.count(c)) {
                window[c]++;
                if (window[c] == need[c]) {
                    valid++;
                }
            }

            while (valid == need.size()) {
                if (right - left + 1 < len) {
                    start = left;
                    len = right - left + 1;
                }

                char d = s[left];
                left++;

                if (need.count(d)) {
                    if (window[d] == need[d]) {
                        valid--;
                    }
                    window[d]--;
                }
            }
        }

        return len == INT_MAX ? "" : s.substr(start, len);
    }
};
```

### 复杂度分析

- 时间复杂度：O(n + m)
  - `n` 是 `s` 长度，`m` 是 `t` 长度。
  - 左右指针都只单向移动。
- 空间复杂度：O(字符集大小)
  - 哈希表保存字符频率。

### 易错点

- `valid` 统计的是“满足数量要求的字符种类数”，不是字符总数。
- 收缩窗口时，如果 `window[d] == need[d]`，要先 `valid--`，再 `window[d]--`。
- 只有 `need` 中的字符才需要维护 `window`。
- 没找到答案时返回空字符串。

### 面试表达

可以这样说：

> 我用 need 记录目标字符需求，用 window 记录当前窗口中目标字符数量。右指针扩张窗口，当某个字符数量满足需求时 valid 加一。当 valid 等于 need 的字符种类数时，说明窗口已经覆盖 t，此时尝试收缩左边界并更新最短答案。收缩时如果某个字符从满足变为不满足，valid 减一。

### 复刷要求

- 一刷：理解 `need/window/valid` 三个变量。
- 二刷：能独立写出收缩窗口逻辑。
- 面试前：能解释它和“找到所有异位词”的区别。

## 本章总结

| 题目 | 类型 | 核心技巧 |
|---|---|---|
| 无重复字符的最长子串 | 可变窗口 | 出现重复就收缩 |
| 找到所有字母异位词 | 固定窗口 | 26 位频率数组 |
| 和为 K 的子数组 | 前缀和 + 哈希 | 不能普通滑窗 |
| 滑动窗口最大值 | 单调队列 | 队头维护最大值 |
| 最小覆盖子串 | 可变窗口 | need/window/valid |

滑动窗口题的判断流程：

1. 题目是否要求连续区间？
2. 窗口长度是固定还是可变？
3. 窗口合法条件是什么？
4. 右指针加入时维护什么？
5. 左指针移出时维护什么？
6. 什么时候更新答案？

最容易混淆的点：

- 有负数的子数组和问题，通常不能直接滑动窗口。
- 固定窗口看长度，可变窗口看条件。
- 单调队列不是普通队列，它维护的是候选答案的单调性。
