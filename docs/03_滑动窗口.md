## 一、滑动窗口知识点回顾

滑动窗口解决的是“连续区间”问题。你可以把它想成拿着一个可伸缩的框，在数组或字符串上从左往右移动。

常见模式：

- 右指针扩张窗口，把新元素加入统计。
- 当窗口不满足条件或已经满足条件时，移动左指针收缩窗口。
- 每次扩张或收缩后更新答案。

注意：滑动窗口通常处理连续子串/子数组；如果题目不是连续区间，先不要硬套。

常用 C++ 容器：

```cpp
unordered_map<char, int> cnt;
vector<int> cnt(26, 0);
deque<int> q;
```

`deque` 是双端队列：

- `q.push_back(x)`：从队尾加入
- `q.pop_back()`：弹出队尾
- `q.pop_front()`：弹出队头
- `q.front()`：访问队头
- `q.back()`：访问队尾

## 1. 无重复字符的最长子串

链接：https://leetcode.cn/problems/longest-substring-without-repeating-characters/

### 题意简述

给一个字符串，找出不含重复字符的最长连续子串长度。

### 生活化理解

像排队进房间，房间里不能有两个同名的人。右边不断进人，一旦发现重名，就从左边开始请人出去，直到房间里没有重复名字。

### 思路

维护窗口 `[left, right]`，用哈希表记录每个字符出现次数。

- 右指针加入字符。
- 如果当前字符次数大于 1，说明重复，移动左指针并减少计数。
- 每次窗口合法后更新最大长度。

### C++ 代码

```cpp
class Solution {
public:
    int lengthOfLongestSubstring(string s) {
        unordered_map<char, int> cnt;
        int left = 0;
        int ans = 0;

        for (int right = 0; right < s.size(); right++) {
            cnt[s[right]]++;

            while (cnt[s[right]] > 1) {
                cnt[s[left]]--;
                left++;
            }

            ans = max(ans, right - left + 1);
        }

        return ans;
    }
};
```

### 复杂度与易错点

- 时间复杂度：O(n)
- 空间复杂度：O(字符集大小)
- 易错点：窗口长度是 `right - left + 1`。

### 面试表达

用滑动窗口维护一个没有重复字符的区间。右指针扩张，出现重复后左指针收缩，直到窗口重新合法。

## 2. 找到字符串中所有字母异位词

链接：https://leetcode.cn/problems/find-all-anagrams-in-a-string/

### 题意简述

在字符串 `s` 中找到所有长度等于 `p`，且字符组成和 `p` 相同的子串起始下标。

### 生活化理解

像拿一个固定长度的透明尺子在字符串上滑动。每次尺子盖住一段字符，就检查这段字符的字母数量是否和目标单词完全一样。

### 思路

因为只包含小写字母，可以用两个长度为 26 的数组：

- `need`：目标字符串 `p` 的字母计数
- `window`：当前窗口的字母计数

窗口大小固定为 `p.size()`，每次加入右边字符，如果窗口过长就移除左边字符。窗口长度刚好时比较两个计数数组。

### C++ 代码

```cpp
class Solution {
public:
    vector<int> findAnagrams(string s, string p) {
        vector<int> ans;
        if (s.size() < p.size()) return ans;

        vector<int> need(26, 0), window(26, 0);
        for (char c : p) need[c - 'a']++;

        int left = 0;
        for (int right = 0; right < s.size(); right++) {
            window[s[right] - 'a']++;

            if (right - left + 1 > p.size()) {
                window[s[left] - 'a']--;
                left++;
            }

            if (right - left + 1 == p.size() && window == need) {
                ans.push_back(left);
            }
        }

        return ans;
    }
};
```

### 复杂度与易错点

- 时间复杂度：O(26n)，可看作 O(n)
- 空间复杂度：O(1)
- 易错点：窗口长度超过 `p.size()` 时要先移除左边。

### 面试表达

用固定长度滑动窗口统计当前子串字符频率，和目标字符串频率数组比较，相同则记录左端点。

## 3. 和为 K 的子数组

链接：https://leetcode.cn/problems/subarray-sum-equals-k/

### 题意简述

给一个整数数组，统计和为 `k` 的连续子数组数量。

### 为什么不是普通滑动窗口

这题虽然是连续子数组，但数组里可能有负数。负数会破坏窗口和的单调性，所以不能用“和大了左移、和小了右移”的普通滑动窗口。

### 生活化理解

你在记流水账，想知道有多少段连续账单刚好合计为 `k`。如果你知道当前位置之前的累计总额是 `sum`，那么只要以前出现过 `sum - k`，中间这一段账单和就是 `k`。

### 思路：前缀和 + 哈希表

维护前缀和 `sum`，以及每个前缀和出现次数 `cnt`。

当前位置前缀和为 `sum`，如果之前存在前缀和 `sum - k`，则这些位置到当前之间的子数组和为 `k`。

初始化：

```cpp
cnt[0] = 1;
```

表示空前缀出现过一次，方便统计从下标 0 开始的子数组。

### C++ 代码

```cpp
class Solution {
public:
    int subarraySum(vector<int>& nums, int k) {
        unordered_map<int, int> cnt;
        cnt[0] = 1;

        int sum = 0;
        int ans = 0;

        for (int x : nums) {
            sum += x;
            if (cnt.count(sum - k)) {
                ans += cnt[sum - k];
            }
            cnt[sum]++;
        }

        return ans;
    }
};
```

### 复杂度与易错点

- 时间复杂度：O(n)
- 空间复杂度：O(n)
- 易错点：必须先查询 `sum - k`，再记录当前 `sum`。
- 易错点：`cnt[0] = 1` 不能忘。

### 面试表达

用前缀和把子数组和转换成两个前缀和之差。遍历时统计以前出现过多少个 `sum - k`，这些位置都能和当前组成和为 k 的子数组。

## 4. 滑动窗口最大值

链接：https://leetcode.cn/problems/sliding-window-maximum/

### 题意简述

给数组和窗口大小 `k`，窗口每次向右滑动一格，返回每个窗口里的最大值。

### 生活化理解

像排队选班长。窗口里只关心最高的人。新来一个更高的人，前面比他矮的人就不可能再成为之后窗口的最高者，可以直接从候选名单里删掉。

### 思路：单调队列

用 `deque<int>` 存下标，并保证队列中对应的值从大到小。

每次处理 `i`：

1. 队头如果已经离开窗口，弹出。
2. 队尾如果比当前值小，弹出，因为它以后不可能成为最大值。
3. 当前下标入队。
4. 当窗口形成后，队头就是最大值下标。

### C++ 代码

```cpp
class Solution {
public:
    vector<int> maxSlidingWindow(vector<int>& nums, int k) {
        vector<int> ans;
        deque<int> q;

        for (int i = 0; i < nums.size(); i++) {
            while (!q.empty() && q.front() <= i - k) {
                q.pop_front();
            }

            while (!q.empty() && nums[q.back()] <= nums[i]) {
                q.pop_back();
            }

            q.push_back(i);

            if (i >= k - 1) {
                ans.push_back(nums[q.front()]);
            }
        }

        return ans;
    }
};
```

### 复杂度与易错点

- 时间复杂度：O(n)
- 空间复杂度：O(k)
- 易错点：队列里存下标，不是存值，这样才能判断是否过期。
- 易错点：窗口形成条件是 `i >= k - 1`。

### 面试表达

用单调递减队列维护窗口最大值候选。队头始终是当前窗口最大值下标，过期元素从队头删，较小元素从队尾删。

## 5. 最小覆盖子串

链接：https://leetcode.cn/problems/minimum-window-substring/

### 题意简述

在字符串 `s` 中找到最短子串，使它包含字符串 `t` 中所有字符及对应数量。

### 生活化理解

像在一长排货架中找最短的一段，里面必须包含购物清单上的所有商品。右边不断扩张直到商品齐了，然后左边尽量收缩，看能不能缩得更短。

### 思路

维护两个哈希表：

- `need`：需要的字符数量
- `window`：当前窗口已有的字符数量

再用 `valid` 表示有多少种字符已经满足要求。

当 `valid == need.size()`，说明窗口已经覆盖 `t`，尝试收缩左边并更新答案。

### C++ 代码

```cpp
class Solution {
public:
    string minWindow(string s, string t) {
        unordered_map<char, int> need, window;
        for (char c : t) need[c]++;

        int left = 0;
        int valid = 0;
        int start = 0;
        int len = INT_MAX;

        for (int right = 0; right < s.size(); right++) {
            char c = s[right];
            if (need.count(c)) {
                window[c]++;
                if (window[c] == need[c]) valid++;
            }

            while (valid == need.size()) {
                if (right - left + 1 < len) {
                    start = left;
                    len = right - left + 1;
                }

                char d = s[left];
                left++;
                if (need.count(d)) {
                    if (window[d] == need[d]) valid--;
                    window[d]--;
                }
            }
        }

        return len == INT_MAX ? "" : s.substr(start, len);
    }
};
```

### 复杂度与易错点

- 时间复杂度：O(n + m)
- 空间复杂度：O(字符集大小)
- 易错点：`valid` 统计的是满足要求的字符种类数，不是字符总数。
- 易错点：收缩左边时，如果某字符从满足变为不满足，要先 `valid--` 再减少计数。

### 面试表达

右指针扩张直到窗口覆盖目标字符，覆盖后左指针尽量收缩并更新最短长度。用 `valid` 记录满足数量要求的字符种类。

## 本章总结

| 题目 | 技巧 |
|---|---|
| 无重复字符的最长子串 | 可变窗口 + 去重 |
| 找到所有字母异位词 | 固定窗口 + 频率数组 |
| 和为 K 的子数组 | 前缀和 + 哈希，不是普通滑窗 |
| 滑动窗口最大值 | 单调队列 |
| 最小覆盖子串 | 可变窗口 + need/window |

复习重点：

- 先判断窗口是否连续。
- 再判断窗口大小固定还是可变。
- 有负数的和问题通常不能直接用普通滑动窗口。
