## 一、知识点回顾

数组是最基础的数据结构，但 Hot100 的数组题并不只是考“会不会遍历”。它通常考这些能力：

| 能力 | 代表题 | 核心问题 |
|---|---|---|
| 维护遍历状态 | 最大子数组和 | 当前状态要不要继承前面 |
| 排序后合并 | 合并区间 | 排序后如何判断重叠 |
| 原地翻转 | 轮转数组 | 如何 O(1) 空间调整数组 |
| 前缀/后缀信息 | 除自身以外数组的乘积 | 如何不用除法获得左右信息 |
| 原地哈希 | 缺失的第一个正数 | 如何用下标表达数字位置 |

普通数组题的常见思考顺序：

1. 能否先用暴力法说明问题？
2. 是否能在一次遍历中维护状态？
3. 排序后问题是否更简单？
4. 是否需要原地修改？
5. 是否能用下标作为“桶”或“哈希位置”？

## 二、基础容器/语法解释

本章常用 C++ 语法：

```cpp
sort(nums.begin(), nums.end());
reverse(nums.begin(), nums.end());
swap(nums[i], nums[j]);
vector<int> ans(n, 1);
```

### 1. sort

```cpp
sort(nums.begin(), nums.end());
```

表示对整个数组升序排序。

如果是二维数组：

```cpp
sort(intervals.begin(), intervals.end());
```

默认先比较 `intervals[i][0]`，如果相同再比较 `intervals[i][1]`。  
所以合并区间里可以直接用默认排序。

### 2. reverse

```cpp
reverse(nums.begin(), nums.end());
reverse(nums.begin(), nums.begin() + k);
```

`reverse(l, r)` 反转的是左闭右开区间 `[l, r)`。

例如：

```cpp
reverse(nums.begin(), nums.begin() + 3);
```

会反转下标 `0, 1, 2`，不包括下标 `3`。

### 3. swap

```cpp
swap(nums[i], nums[j]);
```

交换两个位置的元素。原地哈希题里经常使用。

### 4. vector 初始化

```cpp
vector<int> ans(n, 1);
```

创建长度为 `n` 的数组，所有元素初始值都是 `1`。

```cpp
vector<vector<int>> ans;
ans.push_back(cur);
```

创建二维数组答案，并把一个区间 `cur` 加入答案。

### 5. accumulate

虽然本章代码不一定都用到，但数组求和常用：

```cpp
int sum = accumulate(nums.begin(), nums.end(), 0);
```

需要头文件：

```cpp
#include <numeric>
```

## 三、数组题常见模板

### 一次遍历维护状态

```cpp
int state = 初始值;
for (int x : nums) {
    // 根据 x 更新 state
}
```

适合：

- 最大子数组和
- 股票最佳时机

### 排序后扫描

```cpp
sort(nums.begin(), nums.end());
for (...) {
    // 根据相邻关系处理
}
```

适合：

- 合并区间
- 三数之和

### 前缀/后缀

```cpp
for (int i = 0; i < n; i++) {
    // 处理左侧信息
}
for (int i = n - 1; i >= 0; i--) {
    // 处理右侧信息
}
```

适合：

- 除自身以外数组的乘积
- 接雨水

### 原地哈希

```cpp
while (nums[i] 应该放到别的位置) {
    swap(nums[i], nums[nums[i] - 1]);
}
```

适合：

- 缺失的第一个正数

## 1. 最大子数组和

链接：https://leetcode.cn/problems/maximum-subarray/

标签：数组、动态规划、贪心  
难度：Medium  
重要程度：线性 DP 入门必会

### 题意简述

给定一个整数数组，找出和最大的连续子数组，并返回这个最大和。

关键词：

- 连续子数组：必须连续。
- 可以只选一个元素。
- 数组里可能全是负数。

### 这题考什么

这题考察“当前连续和是否值得继续保留”。

如果前面累计和已经是负数，那么它接到后面只会拖累后面的子数组。  
所以遇到这种情况，应该从当前元素重新开始。

### 生活化理解

想象你在统计连续几天的收益。

如果前几天累计已经亏了 100 元，那么从今天开始算一段新的盈利区间，显然比背着这 100 元亏损继续算更好。

所以每天都问自己一个问题：

> 我要继承前面的收益，还是从今天重新开始？

### 方法一：暴力枚举

枚举所有连续子数组，计算每个子数组和。

伪代码：

```text
for 每个起点 i:
    sum = 0
    for 每个终点 j:
        sum += nums[j]
        更新最大值
```

复杂度：

- 时间复杂度：O(n^2)
- 空间复杂度：O(1)

能帮助理解，但不是面试推荐解。

### 核心思路

定义：

```text
cur = 以当前元素结尾的最大连续子数组和
ans = 全局最大连续子数组和
```

遍历到 `nums[i]` 时有两种选择：

1. 接在前面连续子数组后面：`cur + nums[i]`
2. 从当前元素重新开始：`nums[i]`

所以：

```cpp
cur = max(nums[i], cur + nums[i]);
```

然后：

```cpp
ans = max(ans, cur);
```

### 方法二：动态规划/贪心优化

这题可以叫动态规划，也可以叫贪心式状态维护。

它不需要完整 `dp` 数组，因为当前状态只依赖上一个状态，所以用一个变量 `cur` 即可。

### C++ 题解

```cpp
class Solution {
public:
    int maxSubArray(vector<int>& nums) {
        int cur = nums[0];
        int ans = nums[0];

        for (int i = 1; i < nums.size(); i++) {
            cur = max(nums[i], cur + nums[i]);
            ans = max(ans, cur);
        }

        return ans;
    }
};
```

### 复杂度分析

- 时间复杂度：O(n)
- 空间复杂度：O(1)

### 易错点

- 初始化必须用 `nums[0]`，不能初始化为 0。
- 如果数组全是负数，答案应该是最大的那个负数。
- 题目要求连续子数组，不是子序列。

### 面试表达

可以这样说：

> 我维护 `cur` 表示以当前位置结尾的最大连续子数组和。遍历到当前元素时，要么把当前元素接在前面的连续子数组后面，要么从当前元素重新开始，所以转移是 `cur = max(nums[i], cur + nums[i])`。同时用 `ans` 维护全局最大值。

### 复刷要求

- 一刷：理解为什么负数前缀要丢掉。
- 二刷：能处理全负数。
- 面试前：能讲出 `cur` 的定义。

## 2. 合并区间

链接：https://leetcode.cn/problems/merge-intervals/

标签：数组、排序  
难度：Medium  
重要程度：区间题基础

### 题意简述

给定若干区间，把所有重叠的区间合并，返回合并后的区间列表。

例如：

```text
[1,3] 和 [2,6] 重叠，可以合并成 [1,6]
```

### 这题考什么

这题考察排序后扫描。

如果区间没有排序，你很难判断哪些区间应该先合并。  
按左端点排序后，只需要比较当前区间和结果数组最后一个区间是否重叠。

### 生活化理解

想象你在整理会议室预约表。

如果预约时间是乱的，你很难判断哪些会议冲突。  
但如果先按开始时间排序，就可以从前往后处理：

- 当前会议和上一段时间不冲突，就新开一段。
- 当前会议和上一段时间冲突，就把结束时间延长。

### 方法一：暴力合并

可以反复扫描区间，发现两个重叠区间就合并，直到不能再合并。

问题：

- 实现复杂。
- 时间复杂度高。
- 容易漏边界。

面试不推荐。

### 核心思路

先排序：

```cpp
sort(intervals.begin(), intervals.end());
```

然后维护答案数组 `ans`。

对于当前区间 `cur`：

1. 如果 `ans` 为空，直接加入。
2. 如果 `ans.back()[1] < cur[0]`，说明不重叠，直接加入。
3. 否则说明重叠，更新右端点：

```cpp
ans.back()[1] = max(ans.back()[1], cur[1]);
```

### 方法二：排序 + 一次扫描

这是面试推荐解法。

### C++ 题解

```cpp
class Solution {
public:
    vector<vector<int>> merge(vector<vector<int>>& intervals) {
        sort(intervals.begin(), intervals.end());
        vector<vector<int>> ans;

        for (auto& cur : intervals) {
            if (ans.empty() || ans.back()[1] < cur[0]) {
                ans.push_back(cur);
            } else {
                ans.back()[1] = max(ans.back()[1], cur[1]);
            }
        }

        return ans;
    }
};
```

### 复杂度分析

- 时间复杂度：O(n log n)，主要来自排序。
- 空间复杂度：O(n)，用于保存答案。

### 易错点

- 重叠条件是 `ans.back()[1] >= cur[0]`。
- 不重叠条件是 `ans.back()[1] < cur[0]`。
- 合并时右端点取最大值，不是直接覆盖。

### 面试表达

可以这样说：

> 我先按区间左端点排序。排序后，只需要维护结果数组中的最后一个区间。当前区间如果和最后一个区间不重叠，就加入答案；如果重叠，就更新最后一个区间的右端点为两者右端点最大值。

### 复刷要求

- 一刷：理解为什么要排序。
- 二刷：能准确写出重叠判断。
- 面试前：能手推 `[[1,4],[2,3]]`。

## 3. 轮转数组

链接：https://leetcode.cn/problems/rotate-array/

标签：数组、双指针、原地算法  
难度：Medium  
重要程度：原地数组操作经典题

### 题意简述

给定数组 `nums`，将数组向右轮转 `k` 步。

例如：

```text
[1,2,3,4,5,6,7], k = 3
```

结果：

```text
[5,6,7,1,2,3,4]
```

### 这题考什么

这题考察原地数组变换。

关键点：

- `k` 可能大于数组长度。
- 要尽量 O(1) 额外空间。
- 要理解反转区间的效果。

### 生活化理解

想象一排人整体向右移动。

最后 `k` 个人会被挪到最前面，前面的人整体往后排。

与其一个一个挪，不如用“整体翻转 + 局部翻转”：

1. 先把整队倒过来。
2. 再把前 `k` 个人倒回来。
3. 再把剩下的人倒回来。

### 方法一：额外数组

创建新数组，把每个元素放到新位置：

```cpp
ans[(i + k) % n] = nums[i];
```

复杂度：

- 时间复杂度：O(n)
- 空间复杂度：O(n)

直观但不是最优空间。

### 核心思路

向右轮转 `k` 步后：

```text
后 k 个元素移动到前面
前 n-k 个元素移动到后面
```

三次反转：

```text
原数组：      1 2 3 4 5 6 7
整体反转：    7 6 5 4 3 2 1
反转前 k：   5 6 7 4 3 2 1
反转后 n-k： 5 6 7 1 2 3 4
```

### 方法二：三次反转

这是面试推荐写法。

### C++ 题解

```cpp
class Solution {
public:
    void rotate(vector<int>& nums, int k) {
        int n = nums.size();
        k %= n;

        reverse(nums.begin(), nums.end());
        reverse(nums.begin(), nums.begin() + k);
        reverse(nums.begin() + k, nums.end());
    }
};
```

### 复杂度分析

- 时间复杂度：O(n)
- 空间复杂度：O(1)

### 易错点

- 必须先 `k %= n`。
- `reverse(nums.begin(), nums.begin() + k)` 不包含 `begin() + k`。
- 如果 `k == 0`，代码仍然正确。

### 面试表达

可以这样说：

> 右轮转 k 步相当于把后 k 个元素放到前面。可以通过三次反转原地完成：先整体反转，再反转前 k 个元素，最后反转剩余元素。这样时间 O(n)，额外空间 O(1)。

### 复刷要求

- 一刷：手推三次反转过程。
- 二刷：记住 `k %= n`。
- 面试前：能解释为什么三次反转后顺序正确。

## 4. 除自身以外数组的乘积

链接：https://leetcode.cn/problems/product-of-array-except-self/

标签：数组、前缀积、后缀积  
难度：Medium  
重要程度：前后缀经典题

### 题意简述

给定数组 `nums`，返回数组 `ans`，其中：

```text
ans[i] = 除 nums[i] 以外所有元素的乘积
```

要求：

- 不能使用除法。
- 尽量 O(1) 额外空间。

### 这题考什么

这题考察前缀信息和后缀信息的组合。

对于每个位置 `i`，答案由两部分组成：

```text
i 左边所有数的乘积 * i 右边所有数的乘积
```

### 生活化理解

想象一排人一起贡献力量。

对某个人来说，他不能计算自己，只能计算：

- 左边所有人的贡献。
- 右边所有人的贡献。

所以每个位置的答案，就是左边贡献乘以右边贡献。

### 方法一：左右乘积数组

创建两个数组：

```cpp
left[i] = i 左边所有数乘积
right[i] = i 右边所有数乘积
```

然后：

```cpp
ans[i] = left[i] * right[i];
```

复杂度：

- 时间复杂度：O(n)
- 空间复杂度：O(n)

### 核心思路

可以把 `ans` 数组先当作 `left` 数组使用。

第一遍从左往右：

```cpp
ans[i] = left;
left *= nums[i];
```

此时 `ans[i]` 保存的是 `i` 左边所有数的乘积。

第二遍从右往左：

```cpp
ans[i] *= right;
right *= nums[i];
```

把右边乘积补上。

### 方法二：答案数组承载左乘积 + 变量维护右乘积

这是面试推荐写法。

### C++ 题解

```cpp
class Solution {
public:
    vector<int> productExceptSelf(vector<int>& nums) {
        int n = nums.size();
        vector<int> ans(n, 1);

        int left = 1;
        for (int i = 0; i < n; i++) {
            ans[i] = left;
            left *= nums[i];
        }

        int right = 1;
        for (int i = n - 1; i >= 0; i--) {
            ans[i] *= right;
            right *= nums[i];
        }

        return ans;
    }
};
```

### 复杂度分析

- 时间复杂度：O(n)
- 空间复杂度：O(1)，不计返回数组。

### 易错点

- `left` 和 `right` 初始值都是 1。
- 第一遍 `ans[i] = left` 后，再更新 `left`。
- 第二遍先 `ans[i] *= right`，再更新 `right`。
- 不能使用除法，因为数组中可能有 0。

### 面试表达

可以这样说：

> 每个位置的答案等于它左边所有数的乘积乘以右边所有数的乘积。我先用返回数组保存每个位置左侧乘积，再从右往左维护一个变量 `right` 表示右侧乘积，把它乘到答案中。这样不需要额外左右数组。

### 复刷要求

- 一刷：理解左乘积和右乘积。
- 二刷：能写出 O(1) 额外空间版本。
- 面试前：能解释为什么能处理 0。

## 5. 缺失的第一个正数

链接：https://leetcode.cn/problems/first-missing-positive/

标签：数组、原地哈希  
难度：Hard  
重要程度：数组 Hard 高频题

### 题意简述

给定一个未排序整数数组，找出其中缺失的最小正整数。

要求：

- 时间复杂度 O(n)
- 额外空间 O(1)

例如：

```text
[3,4,-1,1]
```

缺失的第一个正数是 `2`。

### 这题考什么

这题考察原地哈希。

如果数组长度为 `n`，那么缺失的第一个正数只可能在：

```text
1 到 n + 1
```

为什么？

如果 `1..n` 都存在，那么答案就是 `n + 1`。  
否则答案一定是 `1..n` 中缺失的某个数。

### 生活化理解

想象有 `n` 个座位，编号是：

```text
1, 2, 3, ..., n
```

数字 `x` 应该坐到第 `x` 个座位，也就是数组下标 `x - 1`。

我们把每个合法数字都尽量送回自己的座位。  
最后从左往右检查：

- 第 1 个座位不是 1，答案就是 1。
- 第 2 个座位不是 2，答案就是 2。
- ...

第一个没坐对的位置，就是缺失的第一个正数。

### 方法一：哈希集合

用 `unordered_set` 存所有正数，再从 1 开始查找第一个不存在的数。

复杂度：

- 时间复杂度：O(n)
- 空间复杂度：O(n)

不满足 O(1) 额外空间要求。

### 核心思路

把数组本身当哈希表。

规则：

```text
数字 x 应该放在下标 x - 1
```

例如：

```text
1 -> index 0
2 -> index 1
3 -> index 2
```

遍历每个位置 `i`，只要 `nums[i]` 满足：

```cpp
nums[i] >= 1 && nums[i] <= n
```

并且它还没在正确位置：

```cpp
nums[nums[i] - 1] != nums[i]
```

就把它交换到正确位置。

为什么用 `while`？

因为换过来的新数字可能也应该继续被放回自己的位置。

### 方法二：原地哈希

这是面试推荐解法。

### C++ 题解

```cpp
class Solution {
public:
    int firstMissingPositive(vector<int>& nums) {
        int n = nums.size();

        for (int i = 0; i < n; i++) {
            while (nums[i] >= 1 && nums[i] <= n &&
                   nums[nums[i] - 1] != nums[i]) {
                swap(nums[i], nums[nums[i] - 1]);
            }
        }

        for (int i = 0; i < n; i++) {
            if (nums[i] != i + 1) {
                return i + 1;
            }
        }

        return n + 1;
    }
};
```

### 复杂度分析

- 时间复杂度：O(n)
  - 每个数字最多被交换到正确位置一次。
- 空间复杂度：O(1)

### 易错点

- 必须用 `while`，不是 `if`。
- 要判断 `nums[nums[i] - 1] != nums[i]`，否则重复数字会死循环。
- 只处理 `[1, n]` 范围内的数字。
- 最后如果 `1..n` 都在正确位置，答案是 `n + 1`。

### 面试表达

可以这样说：

> 由于长度为 n 的数组中，缺失的第一个正数只可能在 1 到 n+1 之间，所以我把数组下标当作哈希位置。数字 x 应该放在下标 x-1。遍历数组时，把所有合法数字交换到正确位置。最后再次遍历，第一个不满足 `nums[i] == i+1` 的位置，就是缺失的第一个正数。

### 复刷要求

- 一刷：理解为什么答案范围是 `1..n+1`。
- 二刷：能写出 while 条件。
- 面试前：重点手推 `[3,4,-1,1]` 和 `[1,1]`。

## 本章总结

| 题目 | 类型 | 核心技巧 |
|---|---|---|
| 最大子数组和 | 状态维护 | 当前和为负就重新开始 |
| 合并区间 | 排序扫描 | 和结果最后一个区间比较 |
| 轮转数组 | 原地反转 | 三次 reverse |
| 除自身以外数组的乘积 | 前后缀 | 左乘积 + 右乘积 |
| 缺失的第一个正数 | 原地哈希 | 数字 x 放到下标 x-1 |

普通数组题复习重点：

1. 如果题目要求连续区间，先想滑动窗口、前缀和或动态规划。
2. 如果题目涉及区间合并，先排序。
3. 如果题目要求 O(1) 空间，优先考虑原地修改。
4. 如果数字范围和下标有关，考虑原地哈希。
5. 如果需要左右两侧信息，考虑前缀/后缀。
