## 一、知识点回顾

矩阵题本质上是二维数组题。它不难在“会不会写循环”，而难在：

- 行列下标是否清楚。
- 边界是否会走重复。
- 原地修改时是否会污染后续判断。
- 能否利用矩阵的有序性减少搜索范围。

矩阵题常见类型：

| 类型 | 代表题 | 核心技巧 |
|---|---|---|
| 标记行列状态 | 矩阵置零 | 用第一行第一列做标记 |
| 边界收缩遍历 | 螺旋矩阵 | 维护上下左右四个边界 |
| 原地几何变换 | 旋转图像 | 转置 + 反转每一行 |
| 单调性搜索 | 搜索二维矩阵 II | 从右上角或左下角出发 |

做矩阵题时，建议先问自己四个问题：

1. 这题是普通遍历，还是需要控制方向？
2. 需要原地修改吗？
3. 修改当前位置会不会影响后续判断？
4. 行、列是否有单调性可以利用？

## 二、基础容器/语法解释

### 1. 二维 vector 定义

```cpp
vector<vector<int>> matrix;
```

可以理解成“数组里面放数组”。

例如：

```text
matrix = [
  [1, 2, 3],
  [4, 5, 6]
]
```

对应访问：

```cpp
matrix[0][0] == 1
matrix[0][2] == 3
matrix[1][1] == 5
```

### 2. 获取行数和列数

```cpp
int m = matrix.size();
int n = matrix[0].size();
```

- `m`：行数。
- `n`：列数。

在 Hot100 矩阵题中，通常输入矩阵非空，所以可以直接写 `matrix[0].size()`。

### 3. 遍历矩阵

```cpp
for (int i = 0; i < m; i++) {
    for (int j = 0; j < n; j++) {
        // 当前元素是 matrix[i][j]
    }
}
```

含义：

- 外层循环控制行。
- 内层循环控制列。

### 4. 交换两个位置

```cpp
swap(matrix[i][j], matrix[j][i]);
```

用于矩阵转置。

### 5. 反转某一行

```cpp
reverse(matrix[i].begin(), matrix[i].end());
```

表示把第 `i` 行原地反转。

例如：

```text
[1, 2, 3] -> [3, 2, 1]
```

### 6. 结果数组

螺旋矩阵返回一维数组：

```cpp
vector<int> ans;
ans.push_back(matrix[i][j]);
```

`push_back` 表示把元素追加到数组末尾。

## 三、矩阵题常见模板

### 普通二维遍历

```cpp
for (int i = 0; i < m; i++) {
    for (int j = 0; j < n; j++) {
        // 处理 matrix[i][j]
    }
}
```

适合：

- 扫描所有元素。
- 找 0。
- 做状态标记。

### 四边界收缩

```cpp
int top = 0, bottom = m - 1;
int left = 0, right = n - 1;

while (top <= bottom && left <= right) {
    // 从左到右
    // 从上到下
    // 从右到左
    // 从下到上
}
```

适合：

- 螺旋矩阵。
- 按圈遍历。

### 转置矩阵

```cpp
for (int i = 0; i < n; i++) {
    for (int j = i + 1; j < n; j++) {
        swap(matrix[i][j], matrix[j][i]);
    }
}
```

注意 `j = i + 1`，只交换主对角线右上方元素，避免交换两次。

### 右上角搜索

```cpp
int i = 0, j = n - 1;
while (i < m && j >= 0) {
    if (matrix[i][j] == target) return true;
    else if (matrix[i][j] > target) j--;
    else i++;
}
```

适合行列都有序的矩阵。

## 1. 矩阵置零

链接：https://leetcode.cn/problems/set-matrix-zeroes/

标签：数组、哈希表、矩阵
难度：Medium
重要程度：原地标记经典题

### 题意简述

给定一个 `m x n` 矩阵，如果某个元素是 `0`，就把它所在的整行和整列都置为 `0`。

要求尽量原地完成。

### 这题考什么

这题考察原地标记。

关键矛盾是：

如果你一边扫描一边把行列改成 0，后面会分不清这些 0 是原本就有的，还是后来被改出来的。

### 生活化理解

想象你在检查一张表格。如果某个格子出问题，它所在的整行和整列都要标红。

但你不能边检查边直接涂红，因为后面看到红色时，你会误以为它原本就是问题格子。

所以正确做法是：

1. 第一遍只做标记。
2. 第二遍根据标记统一处理。

### 方法一：额外数组记录行列

用两个数组：

```cpp
row[i] = true 表示第 i 行要置零
col[j] = true 表示第 j 列要置零
```

然后再遍历矩阵，根据 `row[i]` 或 `col[j]` 决定是否置零。

复杂度：

- 时间复杂度：O(mn)
- 空间复杂度：O(m + n)

这个方法最直观，适合第一次理解。

### 核心思路

面试推荐用第一行和第一列作为标记数组。

具体含义：

```text
matrix[i][0] == 0 表示第 i 行需要置零
matrix[0][j] == 0 表示第 j 列需要置零
```

但是第一行和第一列本身也可能需要置零，所以要额外记录：

```cpp
bool firstRow = false;
bool firstCol = false;
```

完整流程：

1. 先判断第一行、第一列本身是否有 0。
2. 扫描内部区域，把标记写到第一行和第一列。
3. 根据标记处理内部区域。
4. 最后处理第一行和第一列。

### 方法二：第一行第一列原地标记

这是面试推荐写法。

### C++ 题解

```cpp
class Solution {
public:
    void setZeroes(vector<vector<int>>& matrix) {
        int m = matrix.size();
        int n = matrix[0].size();

        bool firstRow = false;
        bool firstCol = false;

        for (int j = 0; j < n; j++) {
            if (matrix[0][j] == 0) {
                firstRow = true;
            }
        }

        for (int i = 0; i < m; i++) {
            if (matrix[i][0] == 0) {
                firstCol = true;
            }
        }

        for (int i = 1; i < m; i++) {
            for (int j = 1; j < n; j++) {
                if (matrix[i][j] == 0) {
                    matrix[i][0] = 0;
                    matrix[0][j] = 0;
                }
            }
        }

        for (int i = 1; i < m; i++) {
            for (int j = 1; j < n; j++) {
                if (matrix[i][0] == 0 || matrix[0][j] == 0) {
                    matrix[i][j] = 0;
                }
            }
        }

        if (firstRow) {
            for (int j = 0; j < n; j++) {
                matrix[0][j] = 0;
            }
        }

        if (firstCol) {
            for (int i = 0; i < m; i++) {
                matrix[i][0] = 0;
            }
        }
    }
};
```

### 复杂度分析

- 时间复杂度：O(mn)
- 空间复杂度：O(1)

### 易错点

- 不能一边发现 0 一边直接清零整行整列。
- 第一行和第一列要单独记录。
- 内部区域从 `i = 1, j = 1` 开始处理。
- 最后再处理第一行和第一列，顺序不能提前。

### 面试表达

可以这样说：

> 我用第一行和第一列作为标记数组，`matrix[i][0]` 表示第 i 行是否需要置零，`matrix[0][j]` 表示第 j 列是否需要置零。由于第一行和第一列本身也可能需要置零，所以额外用两个布尔变量保存它们的原始状态。最后先处理内部区域，再处理第一行和第一列。

### 复刷要求

- 一刷：先写 O(m+n) 额外空间版本。
- 二刷：理解为什么第一行第一列要单独记录。
- 面试前：能完整说出四步流程。

## 2. 螺旋矩阵

链接：https://leetcode.cn/problems/spiral-matrix/

标签：数组、矩阵、模拟
难度：Medium
重要程度：边界控制经典题

### 题意简述

给定一个矩阵，按照顺时针螺旋顺序返回所有元素。

例如：

```text
1 2 3
4 5 6
7 8 9
```

返回：

```text
[1,2,3,6,9,8,7,4,5]
```

### 这题考什么

这题考察边界控制和模拟能力。

难点不在方向，而在：

- 最后一圈可能只剩一行。
- 最后一圈可能只剩一列。
- 不能重复访问元素。

### 生活化理解

想象你在一个房间里沿墙走一圈。

走完最外圈后，墙往里面缩一层。
再沿新的墙继续走，直到没有区域可走。

### 方法一：访问标记数组

可以创建一个 `visited` 矩阵，按照方向数组模拟前进，遇到边界或访问过的位置就转向。

复杂度：

- 时间复杂度：O(mn)
- 空间复杂度：O(mn)

这个方法直观，但代码较长。

### 核心思路

维护四个边界：

```cpp
int top = 0;
int bottom = m - 1;
int left = 0;
int right = n - 1;
```

每一轮按照四个方向遍历：

1. 从左到右：遍历 `top` 行，然后 `top++`。
2. 从上到下：遍历 `right` 列，然后 `right--`。
3. 从右到左：遍历 `bottom` 行，然后 `bottom--`。
4. 从下到上：遍历 `left` 列，然后 `left++`。

第三步和第四步前要再次检查边界，避免单行或单列时重复遍历。

### 方法二：四边界收缩

这是面试推荐写法。

### C++ 题解

```cpp
class Solution {
public:
    vector<int> spiralOrder(vector<vector<int>>& matrix) {
        int m = matrix.size();
        int n = matrix[0].size();

        vector<int> ans;
        int top = 0;
        int bottom = m - 1;
        int left = 0;
        int right = n - 1;

        while (top <= bottom && left <= right) {
            for (int j = left; j <= right; j++) {
                ans.push_back(matrix[top][j]);
            }
            top++;

            for (int i = top; i <= bottom; i++) {
                ans.push_back(matrix[i][right]);
            }
            right--;

            if (top <= bottom) {
                for (int j = right; j >= left; j--) {
                    ans.push_back(matrix[bottom][j]);
                }
                bottom--;
            }

            if (left <= right) {
                for (int i = bottom; i >= top; i--) {
                    ans.push_back(matrix[i][left]);
                }
                left++;
            }
        }

        return ans;
    }
};
```

### 复杂度分析

- 时间复杂度：O(mn)
- 空间复杂度：O(1)，不计返回数组。

### 易错点

- 第三段从右到左前，要判断 `top <= bottom`。
- 第四段从下到上前，要判断 `left <= right`。
- 更新边界的位置要紧跟对应遍历之后。
- 矩阵不是一定是正方形，可能是 `1 x n` 或 `m x 1`。

### 面试表达

可以这样说：

> 我维护上下左右四个边界，每次按右、下、左、上的顺序遍历当前外圈。走完一条边后收缩对应边界。为了避免最后只剩一行或一列时重复访问，遍历下边和左边前要再次判断边界是否有效。

### 复刷要求

- 一刷：能手推 `3 x 3`。
- 二刷：能处理 `1 x 4` 和 `4 x 1`。
- 面试前：重点记住第三、第四段前的边界判断。

## 3. 旋转图像

链接：https://leetcode.cn/problems/rotate-image/

标签：数组、数学、矩阵
难度：Medium
重要程度：原地矩阵变换经典题

### 题意简述

给定一个 `n x n` 矩阵，将图像原地顺时针旋转 90 度。

要求不能使用额外矩阵。

### 这题考什么

这题考察矩阵原地变换。

关键是找到旋转 90 度和基础操作之间的关系。

### 生活化理解

想象你有一张方形纸。

想让它顺时针转 90 度，可以分两步理解：

1. 沿主对角线翻折。
2. 再左右翻折。

这两个操作对应到矩阵里就是：

1. 转置。
2. 反转每一行。

### 方法一：额外矩阵

新建一个矩阵 `ans`，把每个位置放到旋转后的位置。

对于原位置 `(i, j)`，旋转后位置是：

```text
(j, n - 1 - i)
```

复杂度：

- 时间复杂度：O(n^2)
- 空间复杂度：O(n^2)

直观但不满足原地要求。

### 核心思路

顺时针旋转 90 度可以拆成：

```text
原矩阵
1 2 3
4 5 6
7 8 9

转置后
1 4 7
2 5 8
3 6 9

每行反转后
7 4 1
8 5 2
9 6 3
```

转置代码：

```cpp
for (int i = 0; i < n; i++) {
    for (int j = i + 1; j < n; j++) {
        swap(matrix[i][j], matrix[j][i]);
    }
}
```

每行反转：

```cpp
for (int i = 0; i < n; i++) {
    reverse(matrix[i].begin(), matrix[i].end());
}
```

### 方法二：转置 + 每行反转

这是面试推荐写法。

### C++ 题解

```cpp
class Solution {
public:
    void rotate(vector<vector<int>>& matrix) {
        int n = matrix.size();

        for (int i = 0; i < n; i++) {
            for (int j = i + 1; j < n; j++) {
                swap(matrix[i][j], matrix[j][i]);
            }
        }

        for (int i = 0; i < n; i++) {
            reverse(matrix[i].begin(), matrix[i].end());
        }
    }
};
```

### 复杂度分析

- 时间复杂度：O(n^2)
- 空间复杂度：O(1)

### 易错点

- 转置时 `j` 从 `i + 1` 开始，不能从 0 开始。
- 顺时针旋转是“转置 + 每行反转”。
- 逆时针旋转则不是这个顺序，容易混。
- 题目要求原地修改，不需要返回值。

### 面试表达

可以这样说：

> 顺时针旋转 90 度可以拆成两个原地操作：先沿主对角线转置，再反转每一行。转置负责把行列互换，反转每一行负责把元素调整到顺时针旋转后的位置。

### 复刷要求

- 一刷：手推 `3 x 3`。
- 二刷：能解释为什么 `j = i + 1`。
- 面试前：记住顺时针和逆时针的区别。

## 4. 搜索二维矩阵 II

链接：https://leetcode.cn/problems/search-a-2d-matrix-ii/

标签：数组、二分查找、分治、矩阵
难度：Medium
重要程度：矩阵单调性高频题

### 题意简述

给定一个矩阵，满足：

- 每一行从左到右递增。
- 每一列从上到下递增。

判断矩阵中是否存在目标值 `target`。

### 这题考什么

这题考察如何利用二维单调性排除搜索范围。

如果从左上角开始，右边和下面都更大，无法决定往哪走。
如果从右上角开始：

- 左边都更小。
- 下边都更大。

这样每次都能排除一行或一列。

### 生活化理解

想象你站在一张成绩表的右上角。

当前数字如果太大，说明这一列下面更大，更不可能，所以往左走。
当前数字如果太小，说明这一行左边更小，也不可能，所以往下走。

每走一步，都能扔掉一整行或一整列。

### 方法一：逐行二分

对每一行做二分查找。

复杂度：

- 时间复杂度：O(m log n)
- 空间复杂度：O(1)

可以通过，但没有充分利用列有序。

### 核心思路

从右上角 `(0, n - 1)` 出发：

```text
如果 matrix[i][j] == target，找到。
如果 matrix[i][j] > target，当前列可以排除，j--。
如果 matrix[i][j] < target，当前行可以排除，i++。
```

为什么正确？

- 当前值太大：当前列从上到下递增，下面只会更大，所以这一列不用看。
- 当前值太小：当前行从左到右递增，左边只会更小，所以这一行不用看。

### 方法二：右上角阶梯搜索

这是面试推荐写法。

也可以从左下角出发，逻辑对称：

- 当前值大于目标，往上走。
- 当前值小于目标，往右走。

### C++ 题解

```cpp
class Solution {
public:
    bool searchMatrix(vector<vector<int>>& matrix, int target) {
        int m = matrix.size();
        int n = matrix[0].size();

        int i = 0;
        int j = n - 1;

        while (i < m && j >= 0) {
            if (matrix[i][j] == target) {
                return true;
            } else if (matrix[i][j] > target) {
                j--;
            } else {
                i++;
            }
        }

        return false;
    }
};
```

### 复杂度分析

- 时间复杂度：O(m + n)
- 空间复杂度：O(1)

### 易错点

- 不是从左上角开始。
- 右上角搜索时，大了往左，小了往下。
- 循环条件是 `i < m && j >= 0`。
- 这题和“搜索二维矩阵 I”不同，不能直接把矩阵当作整体一维有序数组。

### 面试表达

可以这样说：

> 我从右上角开始搜索，因为右上角这个位置同时具备两个方向的单调性：往左变小，往下变大。如果当前值大于目标，就排除当前列；如果当前值小于目标，就排除当前行。每一步都能排除一行或一列，所以时间复杂度是 O(m+n)。

### 复刷要求

- 一刷：理解为什么不能从左上角开始。
- 二刷：能区分二维矩阵 I 和 II。
- 面试前：能口述“大了往左，小了往下”。

## 本章总结

| 题目 | 类型 | 核心技巧 |
|---|---|---|
| 矩阵置零 | 原地标记 | 第一行第一列做标记 |
| 螺旋矩阵 | 模拟遍历 | 四边界收缩 |
| 旋转图像 | 原地变换 | 转置 + 每行反转 |
| 搜索二维矩阵 II | 单调性搜索 | 从右上角排除行列 |

矩阵题复习重点：

1. 先明确 `m` 是行数，`n` 是列数。
2. 遍历边界一定写清楚是 `<` 还是 `<=`。
3. 原地修改题要避免“新状态污染旧状态”。
4. 有序矩阵题优先找能稳定排除一行或一列的起点。
5. 面试时要把“为什么不会重复、为什么不会漏”讲清楚。
