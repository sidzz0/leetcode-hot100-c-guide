## 一、知识点回顾

链表题的本质是“改指针”。数组换位置像搬座位，链表换位置像重新接线。做链表题最怕的不是思路复杂，而是顺序写错后断链。

常见能力：

| 能力 | 代表题 | 核心动作 |
|---|---|---|
| dummy 虚拟头节点 | 删除节点、交换节点、合并链表 | 处理头节点变化 |
| 快慢指针 | 找中点、判断环、删除倒数节点 | 让两个指针保持距离差 |
| 反转链表 | 反转链表、K 个一组翻转 | `prev/cur/nxt` 三指针 |
| 哈希映射 | 随机链表复制 | 原节点映射到新节点 |
| 堆/归并 | 合并 K 个链表、排序链表 | 有序链表合并 |
| 哈希表 + 双向链表 | LRU 缓存 | O(1) 查询和调整顺序 |

链表题通用检查：

1. 头节点会不会改变？会就先想 `dummy`。
2. 会不会断链？改 `next` 前先保存后续节点。
3. 是否需要找中点、倒数或环？优先想快慢指针。
4. 是否在构造新链表？优先用 `dummy + cur`。

## 二、基础容器/语法解释

### 1. ListNode 结构

```cpp
struct ListNode {
    int val;
    ListNode *next;
    ListNode() : val(0), next(nullptr) {}
    ListNode(int x) : val(x), next(nullptr) {}
    ListNode(int x, ListNode *next) : val(x), next(next) {}
};
```

- `val`：当前节点的值。
- `next`：指向下一个节点。
- `nullptr`：空指针，表示没有下一个节点。

### 2. 指针访问

```cpp
cur->val;
cur->next;
```

`cur->next` 等价于 `(*cur).next`，表示访问 `cur` 指向的节点里的 `next` 成员。

### 3. dummy 虚拟头节点

```cpp
ListNode dummy(0, head);
ListNode* cur = &dummy;
return dummy.next;
```

`dummy` 是临时放在链表最前面的节点。头节点可能被删除、交换或替换时，用它可以统一处理。

### 4. new 创建节点

```cpp
cur->next = new ListNode(sum % 10);
```

创建一个新节点，并接到当前链表后面。两数相加这类“生成新链表”的题会用到。

### 5. unordered_map 映射节点

```cpp
unordered_map<Node*, Node*> mp;
```

随机链表复制中表示：

```text
原节点地址 -> 新节点地址
```

常见操作：

```cpp
mp[cur] = new Node(cur->val);
mp[cur]->next = mp[cur->next];
mp[cur]->random = mp[cur->random];
```

### 6. priority_queue 小根堆

合并 K 个链表会用小根堆。

```cpp
struct Cmp {
    bool operator()(ListNode* a, ListNode* b) {
        return a->val > b->val;
    }
};

priority_queue<ListNode*, vector<ListNode*>, Cmp> pq;
```

C++ 默认是大根堆，比较器写 `a->val > b->val` 才能让值小的节点优先弹出。

### 7. list 双向链表

LRU 缓存常用：

```cpp
list<pair<int, int>> cache;
unordered_map<int, list<pair<int, int>>::iterator> pos;
```

- `cache` 存 `{key, value}`。
- `pos[key]` 存 key 在链表中的位置。
- 链表头部表示最近使用，尾部表示最久未使用。

常用函数：

```cpp
cache.push_front({key, value});
cache.pop_back();
cache.splice(cache.begin(), cache, it);
```

`splice` 可以把链表中的节点 O(1) 移动到新位置。

## 三、链表题常见模板

### 反转链表模板

```cpp
ListNode* reverse(ListNode* head) {
    ListNode* prev = nullptr;
    ListNode* cur = head;

    while (cur) {
        ListNode* nxt = cur->next;
        cur->next = prev;
        prev = cur;
        cur = nxt;
    }

    return prev;
}
```

### 快慢指针模板

```cpp
ListNode* slow = head;
ListNode* fast = head;

while (fast && fast->next) {
    slow = slow->next;
    fast = fast->next->next;
}
```

### dummy 构造链表模板

```cpp
ListNode dummy;
ListNode* cur = &dummy;

while (...) {
    cur->next = 某个节点;
    cur = cur->next;
}

return dummy.next;
```

## 1. 相交链表

链接：https://leetcode.cn/problems/intersection-of-two-linked-lists/

标签：哈希表、链表、双指针
难度：Easy
重要程度：双指针思想经典题

### 题意简述

判断两条链表是否相交。如果相交，返回第一个相交节点；否则返回 `nullptr`。相交指节点地址相同，不是节点值相同。

### 这题考什么

考察如何消除两条链表的长度差。

### 生活化理解

两个人走两条不同长度的路。走完自己的路后，再去走对方的路。这样两个人走过的总路程都变成 `A 长度 + B 长度`，如果有交点就会相遇，没有交点就一起走到空。

### 方法一：哈希集合

先把链表 A 的所有节点地址放入集合，再遍历链表 B，第一个出现过的节点就是交点。

复杂度：时间 O(m+n)，空间 O(m)。

### 核心思路

两个指针分别从 `headA` 和 `headB` 出发，走到空后切换到另一条链表头部。

```text
p: A -> B
q: B -> A
```

如果相交，二者会在交点相遇；如果不相交，二者会同时变成 `nullptr`。

### 方法二：双指针换路

这是面试推荐写法。

### C++ 题解

```cpp
class Solution {
public:
    ListNode *getIntersectionNode(ListNode *headA, ListNode *headB) {
        ListNode* p = headA;
        ListNode* q = headB;

        while (p != q) {
            p = p ? p->next : headB;
            q = q ? q->next : headA;
        }

        return p;
    }
};
```

### 复杂度分析

- 时间复杂度：O(m+n)
- 空间复杂度：O(1)

### 易错点

- 比较的是节点地址，不是 `val`。
- 不相交时返回的是 `nullptr`。
- 切换链表头部不是重新开始原链表，而是走对方链表。

### 面试表达

可以这样说：两个指针分别走 A+B 和 B+A，总路程相同。如果链表相交，它们会在交点相遇；如果不相交，会同时到达空指针。

### 复刷要求

- 一刷：理解地址相同才是相交。
- 二刷：能解释为什么换路能消除长度差。
- 面试前：能手画不相交情况。

## 2. 反转链表

链接：https://leetcode.cn/problems/reverse-linked-list/

标签：递归、链表
难度：Easy
重要程度：链表基本功第一题

### 题意简述

反转一个单链表，返回反转后的头节点。

### 这题考什么

考察最基础的指针重连。

### 生活化理解

一排人原本都拉着右边的人，现在要改成拉左边的人。改手前必须先记住右边的人是谁，否则后面的队伍会丢。

### 方法一：递归

先递归反转后面的链表，再把当前节点接到反转后链表尾部。能写，但不如迭代稳定。

复杂度：时间 O(n)，空间 O(n)。

### 核心思路

维护三个指针：

- `prev`：已经反转好的部分。
- `cur`：当前节点。
- `nxt`：保存后续节点。

每次先保存 `nxt`，再让 `cur->next = prev`。

### 方法二：迭代三指针

这是面试推荐写法。

### C++ 题解

```cpp
class Solution {
public:
    ListNode* reverseList(ListNode* head) {
        ListNode* prev = nullptr;
        ListNode* cur = head;

        while (cur) {
            ListNode* nxt = cur->next;
            cur->next = prev;
            prev = cur;
            cur = nxt;
        }

        return prev;
    }
};
```

### 复杂度分析

- 时间复杂度：O(n)
- 空间复杂度：O(1)

### 易错点

- 必须先保存 `nxt`。
- 最后返回 `prev`，不是 `head`。
- 空链表和单节点链表自然成立。

### 面试表达

可以这样说：我用 `prev` 保存已反转部分，`cur` 遍历原链表。每次先保存下一个节点，再把当前节点指向前一个节点，最后两个指针前进。

### 复刷要求

- 一刷：背熟三指针模板。
- 二刷：手推 `1->2->3`。
- 面试前：能解释为什么先保存 `nxt`。

## 3. 回文链表

链接：https://leetcode.cn/problems/palindrome-linked-list/

标签：栈、递归、链表、双指针
难度：Easy
重要程度：快慢指针 + 反转链表组合题

### 题意简述

判断链表是否是回文链表。

### 这题考什么

考察快慢指针找中点和反转后半段。

### 生活化理解

检查一串珠子是否左右对称。数组可以两头比较，链表不能倒着走，所以把后半段翻过来再比较。

### 方法一：数组保存值

把链表值放入数组，再用双指针判断数组是否回文。

复杂度：时间 O(n)，空间 O(n)。

### 核心思路

1. 快慢指针找到中点。
2. 反转后半段链表。
3. 前半段和后半段逐个比较。

### 方法二：快慢指针 + 反转后半段

这是面试推荐写法。

### C++ 题解

```cpp
class Solution {
public:
    bool isPalindrome(ListNode* head) {
        ListNode* slow = head;
        ListNode* fast = head;

        while (fast && fast->next) {
            slow = slow->next;
            fast = fast->next->next;
        }

        ListNode* second = reverse(slow);
        ListNode* first = head;

        while (second) {
            if (first->val != second->val) {
                return false;
            }
            first = first->next;
            second = second->next;
        }

        return true;
    }

private:
    ListNode* reverse(ListNode* head) {
        ListNode* prev = nullptr;
        ListNode* cur = head;
        while (cur) {
            ListNode* nxt = cur->next;
            cur->next = prev;
            prev = cur;
            cur = nxt;
        }
        return prev;
    }
};
```

### 复杂度分析

- 时间复杂度：O(n)
- 空间复杂度：O(1)

### 易错点

- 奇数长度时，中间节点不影响比较。
- 比较时以 `second` 是否为空为条件。
- 如果面试官要求恢复链表，需要最后再反转一次后半段。

### 面试表达

可以这样说：我先用快慢指针找到中点，然后反转后半段。之后从头节点和后半段新头节点同时出发比较，全部相同就是回文。

### 复刷要求

- 一刷：写数组版本。
- 二刷：写 O(1) 空间版本。
- 面试前：手推奇数长度链表。

## 4. 环形链表

链接：https://leetcode.cn/problems/linked-list-cycle/

标签：哈希表、链表、双指针
难度：Easy
重要程度：快慢指针基础题

### 题意简述

判断链表是否有环。

### 这题考什么

考察快慢指针判断环。

### 生活化理解

两个人在跑道上跑，一个快一个慢。如果跑道是环形，快的人迟早追上慢的人；如果不是环形，快的人会先到终点。

### 方法一：哈希集合

把访问过的节点地址放入集合，遇到重复节点说明有环。

复杂度：时间 O(n)，空间 O(n)。

### 核心思路

`slow` 每次走一步，`fast` 每次走两步。如果有环，二者会相遇；如果无环，`fast` 会走到空。

### 方法二：快慢指针

这是面试推荐写法。

### C++ 题解

```cpp
class Solution {
public:
    bool hasCycle(ListNode *head) {
        ListNode* slow = head;
        ListNode* fast = head;

        while (fast && fast->next) {
            slow = slow->next;
            fast = fast->next->next;
            if (slow == fast) {
                return true;
            }
        }

        return false;
    }
};
```

### 复杂度分析

- 时间复杂度：O(n)
- 空间复杂度：O(1)

### 易错点

- 循环条件必须是 `fast && fast->next`。
- 判断相遇是地址相同。
- 不要访问空指针的 `next`。

### 面试表达

可以这样说：慢指针走一步，快指针走两步。如果链表有环，快指针一定会在环里追上慢指针；如果无环，快指针会先到空。

### 复刷要求

- 一刷：理解追及模型。
- 二刷：背熟循环条件。
- 面试前：能解释为什么有环一定相遇。

## 5. 环形链表 II

链接：https://leetcode.cn/problems/linked-list-cycle-ii/

标签：哈希表、链表、双指针
难度：Medium
重要程度：快慢指针进阶题

### 题意简述

如果链表有环，返回入环的第一个节点；如果无环，返回 `nullptr`。

### 这题考什么

考察 Floyd 判圈和找环入口。

### 生活化理解

两个人在环里相遇后，让一个人回到起点，另一个人留在相遇点。两个人都每次走一步，再次相遇的位置就是入口。

### 方法一：哈希集合

遍历链表，第一个重复出现的节点就是环入口。

复杂度：时间 O(n)，空间 O(n)。

### 核心思路

1. 快慢指针先在环内相遇。
2. 一个指针回到 `head`，另一个留在相遇点。
3. 两个指针都每次走一步。
4. 再次相遇的位置就是环入口。

### 方法二：Floyd 判圈

这是面试推荐写法。

### C++ 题解

```cpp
class Solution {
public:
    ListNode *detectCycle(ListNode *head) {
        ListNode* slow = head;
        ListNode* fast = head;

        while (fast && fast->next) {
            slow = slow->next;
            fast = fast->next->next;

            if (slow == fast) {
                ListNode* p = head;
                while (p != slow) {
                    p = p->next;
                    slow = slow->next;
                }
                return p;
            }
        }

        return nullptr;
    }
};
```

### 复杂度分析

- 时间复杂度：O(n)
- 空间复杂度：O(1)

### 易错点

- 第一次相遇点不一定是入口。
- 第二阶段两个指针都走一步。
- 无环时返回 `nullptr`。

### 面试表达

可以这样说：先用快慢指针判断是否有环。相遇后，一个指针回头节点，另一个留在相遇点，两者同步前进，再次相遇就是入口。

### 复刷要求

- 一刷：写出代码。
- 二刷：理解第二阶段为什么能找入口。
- 面试前：画图讲清楚。

## 6. 合并两个有序链表

链接：https://leetcode.cn/problems/merge-two-sorted-lists/

标签：递归、链表
难度：Easy
重要程度：dummy 基础题

### 题意简述

合并两个升序链表，返回合并后的升序链表。

### 这题考什么

考察 `dummy + cur` 构造链表，以及归并思想。

### 生活化理解

两队人都已经按身高排好。每次比较两队队首，谁更矮就先加入新队伍。

### 方法一：递归

递归比较两个头节点，较小节点接到后续合并结果前面。

复杂度：时间 O(m+n)，空间 O(m+n)。

### 核心思路

用 `dummy` 保存答案链表前置节点，`cur` 指向答案尾部。每次把较小节点接到 `cur->next`，对应链表往后移动。

### 方法二：迭代归并

这是面试推荐写法。

### C++ 题解

```cpp
class Solution {
public:
    ListNode* mergeTwoLists(ListNode* list1, ListNode* list2) {
        ListNode dummy;
        ListNode* cur = &dummy;

        while (list1 && list2) {
            if (list1->val < list2->val) {
                cur->next = list1;
                list1 = list1->next;
            } else {
                cur->next = list2;
                list2 = list2->next;
            }
            cur = cur->next;
        }

        cur->next = list1 ? list1 : list2;
        return dummy.next;
    }
};
```

### 复杂度分析

- 时间复杂度：O(m+n)
- 空间复杂度：O(1)

### 易错点

- 最后要接上剩余链表。
- 返回 `dummy.next`。
- `cur` 接完节点后要后移。

### 面试表达

可以这样说：我用虚拟头节点构造答案链表，两个指针分别遍历两个升序链表，每次接上较小节点，最后直接接上剩余部分。

### 复刷要求

- 一刷：背熟 dummy 写法。
- 二刷：能写递归和迭代。
- 面试前：能复用到排序链表。

## 7. 两数相加

链接：https://leetcode.cn/problems/add-two-numbers/

标签：递归、链表、数学
难度：Medium
重要程度：链表模拟高频题

### 题意简述

两个链表逆序表示两个非负整数，返回它们相加后的链表。

### 这题考什么

考察链表模拟竖式加法和进位处理。

### 生活化理解

像小学竖式加法，从个位开始加，满十进一。链表正好低位在前，所以从头往后加即可。

### 方法一：转整数

把链表转成整数再相加。这个方法不推荐，因为数字可能溢出。

### 核心思路

维护 `carry`：

```text
sum = 当前 l1 值 + 当前 l2 值 + carry
新节点值 = sum % 10
新进位 = sum / 10
```

循环条件必须包含 `carry`。

### 方法二：链表模拟竖式加法

这是面试推荐写法。

### C++ 题解

```cpp
class Solution {
public:
    ListNode* addTwoNumbers(ListNode* l1, ListNode* l2) {
        ListNode dummy;
        ListNode* cur = &dummy;
        int carry = 0;

        while (l1 || l2 || carry) {
            int sum = carry;

            if (l1) {
                sum += l1->val;
                l1 = l1->next;
            }
            if (l2) {
                sum += l2->val;
                l2 = l2->next;
            }

            carry = sum / 10;
            cur->next = new ListNode(sum % 10);
            cur = cur->next;
        }

        return dummy.next;
    }
};
```

### 复杂度分析

- 时间复杂度：O(max(m,n))
- 空间复杂度：O(1)，不计返回链表。

### 易错点

- 循环条件是 `l1 || l2 || carry`。
- 两条链表长度可能不同。
- 最后一位还有进位时要新建节点。

### 面试表达

可以这样说：链表是逆序存储，所以从头节点开始模拟竖式加法。我维护进位 `carry`，每次创建值为 `sum % 10` 的新节点。

### 复刷要求

- 一刷：手推 `342 + 465`。
- 二刷：处理长度不同。
- 面试前：记住循环条件带 `carry`。

## 8. 删除链表的倒数第 N 个结点

链接：https://leetcode.cn/problems/remove-nth-node-from-end-of-list/

标签：链表、双指针
难度：Medium
重要程度：快慢指针 + dummy 高频题

### 题意简述

删除链表倒数第 `n` 个节点，返回头节点。

### 这题考什么

考察一次遍历定位倒数节点，以及删除头节点的边界处理。

### 生活化理解

两个人排队走，前面的人先走 `n` 步，然后一起走。当前面的人到最后时，后面的人刚好停在要删除节点的前一个位置。

### 方法一：两次遍历

第一次求长度，第二次找到正数第 `len - n + 1` 个节点并删除。

复杂度：时间 O(n)，空间 O(1)。

### 核心思路

用 `dummy` 处理删除头节点。`fast` 先走 `n` 步，然后 `fast` 和 `slow` 一起走，直到 `fast->next` 为空。此时 `slow` 在待删除节点前一个位置。

### 方法二：快慢指针一趟扫描

这是面试推荐写法。

### C++ 题解

```cpp
class Solution {
public:
    ListNode* removeNthFromEnd(ListNode* head, int n) {
        ListNode dummy(0, head);
        ListNode* fast = &dummy;
        ListNode* slow = &dummy;

        for (int i = 0; i < n; i++) {
            fast = fast->next;
        }

        while (fast->next) {
            fast = fast->next;
            slow = slow->next;
        }

        slow->next = slow->next->next;
        return dummy.next;
    }
};
```

### 复杂度分析

- 时间复杂度：O(n)
- 空间复杂度：O(1)

### 易错点

- 必须用 `dummy` 处理删除头节点。
- 循环条件是 `fast->next`。
- 删除的是 `slow->next`。

### 面试表达

可以这样说：我让快指针先走 n 步，再让快慢指针一起走。当快指针到最后一个节点时，慢指针刚好在待删除节点前面。

### 复刷要求

- 一刷：写两次遍历。
- 二刷：写一趟双指针。
- 面试前：手推删除头节点。

## 9. 两两交换链表中的节点

链接：https://leetcode.cn/problems/swap-nodes-in-pairs/

标签：递归、链表
难度：Medium
重要程度：局部指针重连经典题

### 题意简述

每两个相邻节点交换一次，返回交换后的头节点。要求交换节点本身，不是交换值。

### 这题考什么

考察局部指针重连。

### 生活化理解

队伍里每两个人换座位。每换一组，都要先记住后面的队伍接在哪里，再重新牵线。

### 方法一：递归

递归交换前两个节点，再把后续结果接回来。

复杂度：时间 O(n)，空间 O(n)。

### 核心思路

每次处理：

```text
cur -> a -> b -> next
```

交换成：

```text
cur -> b -> a -> next
```

### 方法二：迭代 + dummy

这是面试推荐写法。

### C++ 题解

```cpp
class Solution {
public:
    ListNode* swapPairs(ListNode* head) {
        ListNode dummy(0, head);
        ListNode* cur = &dummy;

        while (cur->next && cur->next->next) {
            ListNode* a = cur->next;
            ListNode* b = cur->next->next;

            a->next = b->next;
            b->next = a;
            cur->next = b;

            cur = a;
        }

        return dummy.next;
    }
};
```

### 复杂度分析

- 时间复杂度：O(n)
- 空间复杂度：O(1)

### 易错点

- 循环条件是 `cur->next && cur->next->next`。
- 交换后 `cur` 移动到 `a`。
- 不能只交换节点值。

### 面试表达

可以这样说：我用 dummy 统一处理头节点变化。每次取 `a` 和 `b` 两个节点，重连成 `b -> a`，再让 `cur` 移动到这一组尾部。

### 复刷要求

- 一刷：画出 `cur/a/b`。
- 二刷：背熟三句重连代码。
- 面试前：手推奇数长度。

## 10. K 个一组翻转链表

链接：https://leetcode.cn/problems/reverse-nodes-in-k-group/

标签：递归、链表
难度：Hard
重要程度：链表综合高频题

### 题意简述

每 `k` 个节点一组翻转链表，不足 `k` 个节点的最后一组保持原顺序。

### 这题考什么

考察分组、断链、局部反转和接回。

### 生活化理解

队伍每 `k` 个人一组调头，不足一组的人保持不动。每组调头前要先知道这组开头、结尾和下一组开头。

### 方法一：递归分组

先判断前 `k` 个节点是否足够，足够就反转前 `k` 个，再递归处理后面。

复杂度：时间 O(n)，空间 O(n/k)。

### 核心思路

每轮：

1. 从 `pre` 出发找第 `k` 个节点 `end`。
2. 不足 `k` 个则结束。
3. 保存 `start = pre->next` 和 `nextGroup = end->next`。
4. 断开当前组，反转，再接回。
5. `pre` 移动到反转后的组尾。

### 方法二：迭代分组反转

这是面试推荐写法。

### C++ 题解

```cpp
class Solution {
public:
    ListNode* reverseKGroup(ListNode* head, int k) {
        ListNode dummy(0, head);
        ListNode* pre = &dummy;

        while (true) {
            ListNode* end = pre;
            for (int i = 0; i < k && end; i++) {
                end = end->next;
            }
            if (!end) {
                break;
            }

            ListNode* start = pre->next;
            ListNode* nextGroup = end->next;
            end->next = nullptr;

            pre->next = reverse(start);
            start->next = nextGroup;
            pre = start;
        }

        return dummy.next;
    }

private:
    ListNode* reverse(ListNode* head) {
        ListNode* prev = nullptr;
        ListNode* cur = head;
        while (cur) {
            ListNode* nxt = cur->next;
            cur->next = prev;
            prev = cur;
            cur = nxt;
        }
        return prev;
    }
};
```

### 复杂度分析

- 时间复杂度：O(n)
- 空间复杂度：O(1)

### 易错点

- 反转前必须确认有 `k` 个节点。
- 要保存 `nextGroup`。
- `start` 反转后变成组尾。
- 断开后才能复用普通反转函数。

### 面试表达

可以这样说：我用 `pre` 指向当前组前一个节点，每次先找组尾。如果够 k 个，就保存下一组头节点，断开当前组，反转后接回链表。

### 复刷要求

- 一刷：先练熟反转链表。
- 二刷：画出 `pre/start/end/nextGroup`。
- 面试前：手推 `k=2`。

## 11. 随机链表的复制

链接：https://leetcode.cn/problems/copy-list-with-random-pointer/

标签：哈希表、链表
难度：Medium
重要程度：深拷贝经典题

### 题意简述

复制一个带 `next` 和 `random` 指针的链表，返回深拷贝后的头节点。

### 这题考什么

考察深拷贝，以及原节点到新节点的映射。

### 生活化理解

像复印一份带朋友关系的名单。先复印所有人，再按原名单补上“朋友指向谁”。

### 方法一：哈希表映射

第一遍创建所有新节点，并记录 `原节点 -> 新节点`。第二遍连接新节点的 `next` 和 `random`。

复杂度：时间 O(n)，空间 O(n)。

### 核心思路

如果 `mp[cur]` 是 `cur` 的复制节点，那么：

```cpp
mp[cur]->next = cur->next ? mp[cur->next] : nullptr;
mp[cur]->random = cur->random ? mp[cur->random] : nullptr;
```

### 方法二：节点穿插法

把复制节点插到原节点后面，设置 random 后再拆开。空间更优，但代码更容易写错。秋招面试时间紧时，哈希表版本更稳。

### C++ 题解

```cpp
class Solution {
public:
    Node* copyRandomList(Node* head) {
        if (!head) {
            return nullptr;
        }

        unordered_map<Node*, Node*> mp;
        Node* cur = head;

        while (cur) {
            mp[cur] = new Node(cur->val);
            cur = cur->next;
        }

        cur = head;
        while (cur) {
            mp[cur]->next = cur->next ? mp[cur->next] : nullptr;
            mp[cur]->random = cur->random ? mp[cur->random] : nullptr;
            cur = cur->next;
        }

        return mp[head];
    }
};
```

### 复杂度分析

- 时间复杂度：O(n)
- 空间复杂度：O(n)

### 易错点

- 是深拷贝，不能复用原节点。
- `random` 可能是 `nullptr`。
- `random` 可能指向自己。
- 先创建节点，再连接指针。

### 面试表达

可以这样说：我用哈希表建立原节点到新节点的映射。第一遍创建所有复制节点，第二遍根据映射补上 `next` 和 `random`。

### 复刷要求

- 一刷：写哈希表版本。
- 二刷：理解穿插法。
- 面试前：能解释深拷贝。

## 12. 排序链表

链接：https://leetcode.cn/problems/sort-list/

标签：链表、双指针、分治、排序、归并排序
难度：Medium
重要程度：链表归并排序经典题

### 题意简述

对链表进行升序排序。

### 这题考什么

考察链表上的归并排序。

### 生活化理解

把一长队人不断从中间拆开，分别排好后再按顺序合并。

### 方法一：转数组排序

把链表值放入数组，排序后再写回链表。

复杂度：时间 O(n log n)，空间 O(n)。

### 核心思路

1. 快慢指针找中点。
2. 断开链表。
3. 递归排序左右两段。
4. 合并两个有序链表。

### 方法二：归并排序

这是面试推荐写法。

### C++ 题解

```cpp
class Solution {
public:
    ListNode* sortList(ListNode* head) {
        if (!head || !head->next) {
            return head;
        }

        ListNode* slow = head;
        ListNode* fast = head->next;
        while (fast && fast->next) {
            slow = slow->next;
            fast = fast->next->next;
        }

        ListNode* mid = slow->next;
        slow->next = nullptr;

        return merge(sortList(head), sortList(mid));
    }

private:
    ListNode* merge(ListNode* a, ListNode* b) {
        ListNode dummy;
        ListNode* cur = &dummy;

        while (a && b) {
            if (a->val < b->val) {
                cur->next = a;
                a = a->next;
            } else {
                cur->next = b;
                b = b->next;
            }
            cur = cur->next;
        }

        cur->next = a ? a : b;
        return dummy.next;
    }
};
```

### 复杂度分析

- 时间复杂度：O(n log n)
- 空间复杂度：O(log n)，递归栈。

### 易错点

- `fast = head->next` 可以让 `slow` 停在前半段尾部。
- 必须 `slow->next = nullptr` 断链。
- 合并函数复用合并两个有序链表模板。

### 面试表达

可以这样说：链表排序适合归并排序，因为合并两个有序链表只需要改指针。我用快慢指针找中点，断开后递归排序两段，再合并。

### 复刷要求

- 一刷：复习合并两个有序链表。
- 二刷：写出找中点和断链。
- 面试前：讲清楚 O(n log n)。

## 13. 合并 K 个升序链表

链接：https://leetcode.cn/problems/merge-k-sorted-lists/

标签：链表、分治、堆、归并排序
难度：Hard
重要程度：堆和分治高频题

### 题意简述

合并 `k` 个升序链表，返回一个升序链表。

### 这题考什么

考察多路归并。

### 生活化理解

有 K 条已经排好队的队伍。每次从所有队首中选最小的人加入新队伍，小根堆就是自动帮你找当前最小队首的工具。

### 方法一：顺序合并

从第一条链表开始，依次和后面的链表合并。

复杂度最坏 O(kN)，其中 `N` 是总节点数。

### 核心思路

小根堆里维护每条链表当前头节点：

1. 所有非空头节点入堆。
2. 弹出最小节点接到答案后面。
3. 如果这个节点有后继，后继入堆。
4. 堆空时结束。

### 方法二：小根堆多路归并

这是面试推荐写法之一。分治合并也建议掌握。

### C++ 题解

```cpp
class Solution {
public:
    struct Cmp {
        bool operator()(ListNode* a, ListNode* b) {
            return a->val > b->val;
        }
    };

    ListNode* mergeKLists(vector<ListNode*>& lists) {
        priority_queue<ListNode*, vector<ListNode*>, Cmp> pq;

        for (ListNode* node : lists) {
            if (node) {
                pq.push(node);
            }
        }

        ListNode dummy;
        ListNode* cur = &dummy;

        while (!pq.empty()) {
            ListNode* node = pq.top();
            pq.pop();

            cur->next = node;
            cur = cur->next;

            if (node->next) {
                pq.push(node->next);
            }
        }

        return dummy.next;
    }
};
```

### 复杂度分析

- 时间复杂度：O(N log k)
- 空间复杂度：O(k)

### 易错点

- C++ `priority_queue` 默认是大根堆。
- 比较器写 `a->val > b->val` 才是小根堆。
- 空链表不要入堆。

### 面试表达

可以这样说：我用小根堆维护每条链表当前最小节点。每弹出一个节点就接到答案链表后，如果它有下一个节点，就把下一个节点放入堆。

### 复刷要求

- 一刷：写堆版本。
- 二刷：补分治版本。
- 面试前：能解释 `O(N log k)`。

## 14. LRU 缓存

链接：https://leetcode.cn/problems/lru-cache/

标签：设计、哈希表、链表、双向链表
难度：Medium
重要程度：设计题高频题

### 题意简述

设计 LRU 缓存，支持 `get` 和 `put`，要求平均 O(1)。容量满时，淘汰最久未使用的数据。

### 这题考什么

考察 `unordered_map + list` 的组合。

### 生活化理解

像桌面文件。最近用过的文件放最上面，桌面满了就丢掉最底下最久没用过的文件。

### 方法一：哈希表 + 时间戳

查询可以 O(1)，但淘汰时要扫描所有 key 找最久未使用，不能满足 O(1)。

### 核心思路

双向链表维护使用顺序：

```text
头部 = 最近使用
尾部 = 最久未使用
```

哈希表维护：

```text
key -> 链表节点位置
```

每次访问或更新 key，就移动到链表头部；容量满时删除尾部。

### 方法二：unordered_map + list

这是面试推荐写法。

### C++ 题解

```cpp
class LRUCache {
private:
    int cap;
    list<pair<int, int>> cache;
    unordered_map<int, list<pair<int, int>>::iterator> pos;

    void moveToFront(int key) {
        auto it = pos[key];
        cache.splice(cache.begin(), cache, it);
        pos[key] = cache.begin();
    }

public:
    LRUCache(int capacity) {
        cap = capacity;
    }

    int get(int key) {
        if (!pos.count(key)) {
            return -1;
        }
        moveToFront(key);
        return pos[key]->second;
    }

    void put(int key, int value) {
        if (pos.count(key)) {
            pos[key]->second = value;
            moveToFront(key);
            return;
        }

        if (cache.size() == cap) {
            int oldKey = cache.back().first;
            cache.pop_back();
            pos.erase(oldKey);
        }

        cache.push_front({key, value});
        pos[key] = cache.begin();
    }
};
```

### 复杂度分析

- `get` 时间复杂度：O(1)
- `put` 时间复杂度：O(1)
- 空间复杂度：O(capacity)

### 易错点

- `get` 命中后也要移动到头部。
- `put` 更新已有 key 也要移动到头部。
- 淘汰尾部时，要同时从哈希表删除。
- 哈希表里存的是链表迭代器。

### 面试表达

可以这样说：LRU 需要 O(1) 查询和 O(1) 调整使用顺序，所以用哈希表定位节点，用双向链表维护最近使用顺序。访问节点时移动到头部，容量满时删除尾部。

### 复刷要求

- 一刷：理解 map 和 list 各自负责什么。
- 二刷：写 STL `list` 版本。
- 面试前：能解释 `splice`。

## 本章总结

| 题目 | 类型 | 核心技巧 |
|---|---|---|
| 相交链表 | 双指针 | 走完换路，消除长度差 |
| 反转链表 | 指针反转 | `prev/cur/nxt` |
| 回文链表 | 快慢指针 | 找中点 + 反转后半段 |
| 环形链表 | 快慢指针 | 快指针追慢指针 |
| 环形链表 II | Floyd 判圈 | 相遇后一个指针回头 |
| 合并两个有序链表 | 归并 | dummy 接小节点 |
| 两数相加 | 模拟 | 进位 carry |
| 删除倒数第 N 个节点 | 快慢指针 | 快指针先走 n 步 |
| 两两交换节点 | 局部重连 | `cur -> b -> a` |
| K 个一组翻转 | 分组反转 | 找组尾、断开、反转、接回 |
| 随机链表复制 | 哈希映射 | 原节点映射新节点 |
| 排序链表 | 归并排序 | 找中点、断链、合并 |
| 合并 K 个链表 | 小根堆 | 多路归并 |
| LRU 缓存 | 设计 | 哈希表 + 双向链表 |

链表题复习重点：

1. 先画节点和指针，再写代码。
2. 头节点可能变化时，优先用 `dummy`。
3. 改 `next` 前先保存后续节点。
4. 倒数、中点、环，优先想快慢指针。
5. 复杂链表题通常是基础动作组合。
